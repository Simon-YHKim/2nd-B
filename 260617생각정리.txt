1. 지금 내가 생각하는 2ndb 앱의 시스템의, 나자신 알아가기는 7가지 축으로 가려고 해.
 - 지금의 나 : 브레인 트리니티 형식을 빌려서, 현재의 나의 상태에 대해서 조사하는 도구.  형식을 그대로 따라가되, AI를 통해 핵심을 질문하고 되묻는 방법으로 빌드업 해가는 방식. 한번의 질문만으로 확정하지 않고, 성격검사들이 그러하듯 교묘하게 반복되는 질문으로 결과의 정합성을 높힌다.
 - 회상(북두칠성의 네번째) : 각 시기별(유년기, 유아기, 유치원, 초등학교, 중학교 ... 20 초반, 20대 중반, 20대 후반... 사용자의 오늘날 까지) drill-down 질문을 통해 사용자가 왜 이런 사람이 됐는지, 어떤 사람인지에 대해 상세하게 파악하는 도구. 마찬가지로 교묘하게 반복되는 질문으로 결과의 정합성을 높힌다.
 - 보여지는 나(북두칠성의 다섯번째) : peer review의 한 형태로, 기본적인 peer review 형태를 따라가되, 사용자의 만의 생각에 대해 다른 사람의 시선이 어떤지 확인하는것. 단순히 다른 사람의 시선을 파악하는 역할도 함께한다. 하지만 한두사람만의 의견으로는 전체적인 외부의 시선이라 볼수 없으며, 마주하는 형태(직장 동료, 친구, 대학친구, 가족 등등)에 따라 시선이 다를수 있다. 카테고리를 나눠서 각 의견의 경향성 및 정합성을 얻어내는것이 관건.  질문지는 사용자별 최적화 되어야 하며, 설문지를 답변하는 peer의 입장에서는 쉽게 할수 있어야 한다. 즉, 매우 그렇다/그렇다/보통/아니다/매우아니다 같은 형태를 활용하는것도 방법.

2. 산출과 관련해서는,
 - 연결축을 찾아내고, 학술적으로 검증된 자료를 통해 개인의 특성, 성향, 성격 등등에 대해 산출해낸다. 각 데이터별 관계성을 파악할수 있도록 태깅, 등의 방법이 중요하다 
 - 항상 정형화된 질문 순서와 의도는 지키되 상황에 따라 필요한 부분을 AI가 재질문을 한다. 
 - 각 단계를 통과할때는, 검증된방법으로 정량화 하여 신뢰도가 높은 점수 까지 도달할때까지 사용자를 deep dive drill 한다. 점수를 어떻게 측정할지는 고민해야할 문제.

3. AI 엔진의 설정은,
 - 사용자를 파악할때, 어떤 AI를 활용하더라도 높은 품질을 유지하도록 하네스 엔지니어링이 중요함.
 - WiKi, RAG 시스템에 착안하여, 사용자 데이터 기반으로 사용자에게 맞는, 필요한, 궁금해할, 답변을 해야한다.
 - Clipper AI 엔진은 : 기존에 내가 제공한 클립핑 양식을 따라서 태깅이 잘 될수 있도록 한다.
 - Wiki AI 엔진은 : 태깅된 데이터들을 카테고리화 하여 정리하고, AI가 쉽게 읽을수 있는 형태로 구조화 한다.(graphify 그래프 구조 산출)

7. 이 앱을 사용할때 AI와 나의 상호작용에 대해 고민하면,
 - oth brain(me,myself&I) : 철학, 역할의 집합체.
 - 1st brain(물리적 뇌) : 뉴런들의 네트워크
 - 2nd brain(LLM Wiki) : 내 파일, 데이터, 지식, 정보들의 유기적인 조합의 네트워크
 - 3rd brain(LLM AI) : 인공 뉴럴 네트워크. 확장하고, 새로운것영역으로 넓히고, 새로운것을 배우게 도와주는 역할
따라서, 우리 앱의 역할은 1st brain을 위해서 2nd brain를 만들고, 3rd brain을 연결하여 나를 돕는 전용 비서같은 AI를 만들고, 이 AI를 통해 북극성을 향해 정진할수 있게 돕는 것.

8. 브레인 트리니티의 구조에 대해서 고민하면,
1) 철학(북극성) : 내가 나로서 살아가는 이유와 방향
 - 자아, 가치, 비전 즉, 내가 누구이고, 무엇을 사랑하며, 어디로 가고 있는가
 - who(나는 누구인가_identity), Why(왜 사는가_purpose), What(무엇을 사랑하고 중요시 여기는가_values), Where(어디로 나아가는가_goal), How(어떻게 살 것인가_Principles)
 - 영역 : 커리어, 재정, 성장, 가족, 관계, 환경, 건강, 오락
2) 역할 : 내가 살아가는 다양한 장의 무대(하나의 철학이 분화되어 표현되는 자리 들, 정체성). 개인에 따라 여러가지일 수 있음.
 - Roles(북두칠성의 첫번째) : 역할과 목표.(e.g. Lean Production 엔지니어, 사업부의 니즈에 따라 적기에 생산성 개선활동을 수행하여 필요한 시기에 필요한 Capa를 제공) 
 - Action(북두칠성의 두번째) : 역할별 하위 프로젝트
 - Knowledge(북두칠성의 세번째) : 프로젝트별 하위 작업

9. 우리 앱이 운용되는 구조를 고민하면 :
 - 바깥 세상의 지식이 1st barin과 3rd brain을 통해 유입된다.
 - Workspace: 지식이 심어지고, 자라나고, 살아나는 영토
    ┠1st brain : My note → human ingest(skills : brain-add-seed, brain-brainify, brain-connect, brain-distill) → 깊은 이해 
    └3rd brain : LLM Wiki → AI Ingest(skills : ingest, lint, query, graphify) → 얕은 이해
 - Workflow : 깊은 이해 + 얕은 이해 → 통합/증강된 이해 → 새로운 지식 창출 → 내 역할들 에게 영향(Value-adding)
 - My note : Atlas, Brain, Calendar, Distill, Efforts, Note, Log, Journal, Planner, Reading, Meeting, Project, Task, Zettel, Literature, maps + 기타 외부 데이터.
 - LLM Wiki : Clipper 엔진, LLM Wiki 엔진으로 가동


10. 기타 : 
 - 사용자가 사용시간이 길어짐에 따라 기존 데이터와 다른 방향으로 신규 데이터들 주입할수도 있다. 데이터가 쌓여서 최근 경향성이 달라진다고 판단될때는 사용자에게 브레인 트리니티를 점검하라고 한다. 이 과정에서 AI가 먼저 산출/수정한 데이터를 보여주고, 사용자와 소통하고 충분한 합의가 이뤄졌을때 승인 하는 형태로 데이터를 변형해간다.
 - 외부 앱 연동성을 확대하여 사용자 데이터를 얻을수 있게 하고, 사용자를 더욱 자세히 알수 있게 한다.
 - 모든 북두칠성이 켜지면 북극성이 더 밝아진다. 북극성은 북두칠성보다 밝다.
 - 북두칠성의 각 별은 데이터의 품질 평가를 통해 밝기를 차등 한다. 5단계 정도로.
 - 데이터가 가치를 얻어 정보를 넘어 지혜로 발전하는 과정은, 개인화의 정도에 따라 나눠지며 그 단계가 올라감에 따라 가치가 상승한다. 단계의 순서는 Data(원시적이고 구조화 되지 않은 데이터. 독립적임) → Information(data*pattern_맥락 안에서 조직화 하여 해석, 의미를 부여) → Knowledge(Information*Meaning_정보에 맥락(관계, 법칙)이 부여된 것, 응용이 가능한 상태) → Wisdom(Knowledge*Action_지식을 기반으로 행동으로 옮길수 있는 상태. 상황에 맞는 결정을 내리는데 필요한 직관과 통찰력)
 - Bloom's Taxonomy 참고.
 - 현재 북두칠성의 6번째, 7번째가 비어있음. 고민 필요.

11. 참고자료 : 
[개인지식관리의 구성요소]

1. 시스템 System (Organization)
   : 정보와 지식이 관리되는 전체적인 구성요소와 체계
   ■ 제 1의 두뇌 (실시간 사고, 판단)
     - 정의:
       • 생물학적 두뇌, 즉각적인 인지와 사고
       • 정보 처리, 창의적 사고, 문제 해결
       • 외부 세계와 직접 상호작용
     - 특징:
       • 제한된 용량, 단기기억의 한계
       • 집중력, 통찰에 대한 인지적 기능
       • 경험과 학습을 통한 관점 형성
   ■ 제 2의 두뇌 (지식 저장고, 작업공간)
     - 정의:
       • 개인지식관리 시스템, 도구를 활용해 지식 저장, 조직화
       • 제 1 두뇌에서 처리된 데이터 저장
     - 특징:
       • 제한 없는 저장 용량
       • 정보 체계적으로 정리, 연결
       • 빠른 검색 및 재활용

2. 프로세스 Processes (Workflow)
   : 외부세계의 데이터와 정보를 수집하여 개인지식으로 만들어 새로운 가치로 탄생하는 전체 과정
   [단계 1] 정보/지식의 수집
   (외부 세계의 정보를 관찰, 개인 지식화 단계 / 잠재 가치의 발견)
   1. 수집 (Collect)
      - 외부 세계를 관찰, '가치'를 느낄 정보와 지식의 수집
      - 주요 키워드: 관찰, 영감, 연결
   2. 분류 (Categorize)
      - 수집된 정보의 체계적인 정리와 분류
      - 시스템에 저장 (= 장기기억)
      - 주요 키워드: 체계화된 저장
   [단계 2] 정보/지식의 활용
   (수집된 정보에 새로운 통찰이나 결과물 창출 / 실질적인 가치의 실현)
   3. 연결 (Connect)
      - 정보의 검색과 인출(Retrieve), 새로운 연결성을 발견하여 확장
      - 주요 키워드: 빠른 검색 및 활용
   4. 창조 (Create)
      - 새로운 지식, 아이디어, 결과물의 창출
      - 주요 키워드: 새로운 지식의 창조, 연결

※ 프로세스 흐름: 1. 수집 ➔ 2. 분류 ➔ 3. 연결 ➔ 4. 창조 순으로 순환 및 연결됨
