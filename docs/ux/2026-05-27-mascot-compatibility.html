05검토 항목 5 — 시그니처·외형 디테일 수정

Q5 · Visual Language Compatibility
GBC 8-bit 픽셀 스타일이 다크 sky 미니멀 UI 와 한 화면에 살 수 있는가?
🛑 STOP — 시각 언어 충돌 큼. 재해석 필요

현재 앱은 현대적 미니멀(부드러운 그라데이션, vector 아이콘, anti-aliased 글리프) 인데 마스코트는 레트로 픽셀(no anti-aliasing, hard edges, 32×32). 같은 화면에 놓으면 둘 다 가짜로 보일 위험.

3가지 진로

옵션요약장점단점
A. 격리 사용 픽셀 그대로, /loading + /index 빈 상태 + Easter egg 에만 등장 마스코트 v1.0 그대로 사용, GPT 생성 비용 0 추가 22개 중 3-4개 화면만 노출 → 마스코트 효과 제한적
B. 재해석 (vector 일러스트) ⭐ 같은 캐릭터·컬러·시그니처를 vector minimal 로 다시 그림 전 화면에 사용 가능, 다크 sky 미니멀과 자연 융합 일러스트 작업 추가 — Figma/Stitch/Midjourney v6 niji
C. 하이브리드 픽셀 = 마스코트 인트로(/loading), Vector = 인앱 일상 사용 레트로 시그니처 + 일상 통합 모두 확보 두 자산 셋 관리 — 캐릭터 일관성 부담

권장: B. 픽셀 컨셉은 매력적이지만 우리 앱의 미적 정체성(현대적 다크 sky)과 갈등이 큽니다. 마스코트 컨셉(컬러+성격+시그니처 아이템+quote)을 유지한 채 vector minimal 로 재해석하면 22개 화면 모두에 자연스럽게 박혀요.

vector 재해석 추가 가이드

  • 32×32 → 64×64 또는 SVG. 인터페이스 크기에 따라 64/96/128 출력
  • anti-aliased + 부드러운 그림자 + 다크 sky bg 위에서 빛나는 미세 glow
  • 퍼시즈 톤 — 라벨/태그/명찰은 그대로 유지(시그니처가 캐릭터 식별의 핵심)
  • vector 라인은 1.5pt — 다크 bg 위에서 가독성 확보

07가장 큰 결정 포인트 — Brain Stack vs Brain Trinity

마스코트 컨셉은 9-node Brain Stack 시스템(0~3rd Brain + 5 workflow nodes)을 가정해요. 우리 앱은 4-area Brain Trinity(건강/앱/뇌/재정)를 이미 구현했고요. 두 모델이 같은 단어("Brain")를 쓰는데 다른 차원이라 충돌 가능.

3가지 화해 안

  • 가. Trinity 유지 + Stack 마스코트 적용 — Stack은 메타포(시스템 내부 직군), Trinity는 사용자 대시보드. 마스코트들은 "Trinity 4영역의 일꾼"으로 재정의. 권장.
  • 나. Trinity → Brain Stack 으로 교체 — 큰 리팩토링. trinity.tsx 폐기. 데드라인 26.08.17 고려하면 risky.
  • 다. 두 시스템 병존 — Trinity = 사용자 대시보드, Brain Stack = 분석 시스템 내부. 사용자는 Trinity 만 보고, Brain Stack은 v0.22 분석 시스템 설계도(이번에 받은 자료)에 박힘. 가장 깔끔.

제 권장: 다. v0.22 분석 시스템 설계도의 4-Layer LLM-agnostic + Behavior Taxonomy + Ledger/Voice 가 Brain Stack의 내부 구현이 되고, 마스코트는 그 시스템의 직군을 의인화. Trinity는 사용자 대시보드로 남음. 두 컨셉 모두 살리면서 충돌 회피.
