01한 줄 요약

전체 시각 골격(다크 sky 팔레트, LoadingScreen 25문구, 그래프 랜딩)은 잘 잡혔는데, 세포-톤 브랜드 보이스가 22개 중 17개 화면에서 빠져 있고, AppNav 가 9개 화면에 안 붙어 있어서 발견성 스토리가 무너져요. 즉시 손볼 critical은 3가지: (1) 헥스 리터럴 두 곳 (랜딩·로그인), (2) 세포 톤 누락 17 화면, (3) AppNav 커버리지 9 화면.

03잘된 점 (Top 5)

WIN 1LoadingScreen 25문구

세포 톤이 가장 완벽하게 구현된 모범 사례. Soma → Neuron → Cortex → Cerebrum 단계 진행이 카피와 시각이 모두 맞춰져 있어요.

LoadingScreen → 그래프 랜딩 전환이 로고 중심 좌표가 정확히 맞아서 끊김 없음.

WIN 3AppNav 3-rail 디자인
src/components/ui/AppNav.tsx

DAILY · SELF-KNOWLEDGE · MANAGE 3행 구조로 "페이지 간 연속성 없음" 문제를 개념적으로 해결.

WIN 4CI 가드레일 그린

Lexicon, constraints (C1-C12), i18n parity (99 keys × 8 namespace), LLM boundary 전부 통과.

WIN 5+not-found 친절함

"이 길은 아직 만들어지지 않았어요" — 톤 살아 있음.

06XPRIZE 심사 가장 critical 3가지

JUDGE-CRIT 1세포 톤이 17개 화면에서 누락

DESIGN.md(SoT)가 capture, inbox, jarvis, wiki, insights, audit, interview, big-five, attachment, mbti를 마이그레이션 타깃으로 명시. 심사관이 capture.tsx:83-84의 "Captured as Article: ..." 같은 일반 SaaS 문구를 마주하는 순간, "내 머릿속의 열일하는 세포들" 브랜드 약속이 첫 액션에서 무너져요.

JUDGE-CRIT 2헥스 리터럴이 디자인 시스템 위반

index.tsx:199,223 + sign-in.tsx:32-40. 위반 자리가 랜딩 화면 + 메인 인증 입구라 심사관이 둘 다 거쳐 갈 위치. QA 돌리는 리뷰어가 즉시 잡을 항목.

JUDGE-CRIT 3AppNav 커버리지 갭이 발견성 무너뜨림

22개 중 9개 (/persona, /big-five, /attachment, /mbti, /interview, /permissions 등)에 AppNav 없음. AppNav.tsx 주석에 인용된 사용자 지시 "페이지 간에 연속적이다 즉 네비게이팅이 직관적으로 쉽게 된다는 느낌이 없어"가 절반만 해결됨. 심사관이 Big Five 먼저 풀면 /persona로 가는데(good), /persona 먼저 들어가면 빈 상태 + 6버튼 메뉴(AppNav 중복)와 만남. 같은 목적지 다른 UI — 발견성 스토리 붕괴.
