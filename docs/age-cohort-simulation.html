글 읽는 영유아부터 90대까지 — 50명의 가상 사용자가 앱을 만나는 법

10개 연령대 × 5명(직업·배경 다양) = 50 페르소나를, 실제 코드 흐름 기준으로 A→Z 시뮬레이션한 결과. 각 인물의 가입 통과/차단 · 온보딩 이해 · 끌리는 축 · 마찰/접근성 · 이탈 위험 · 평결을 추적했다. 자매 문서: 사용자 여정 시뮬레이션(A→Z).

근거: 5개 병렬 시뮬레이터(6–17 · 18–39 · 40–59 · 60–79 · 80–99). 코드 실측 — 배경 space950 #070A18, 보조텍스트 mistGray #8D98B8(대비 ~3.5:1, WCAG AA 미달), 픽셀폰트 NeoDunggeunmo, 탭 라벨 10px, 생년월일 YYYY-MM-DD 텍스트입력, 일기 Lv3 잠금, 음성입력 없음.

01전 연령 공통 8대 발견

밴드를 가로질러 반복된 패턴. 위로 갈수록 영향 범위가 넓다.

1
일기 Lv3 잠금 = 단일 최대 이탈 트리거. 20대~80대까지, 가장 자연스러운 진입(감정 토로·회고·일기)을 첫날 차단. 재수생·육아맘·간호사·소방관·농부·은퇴교사·독거 어르신 — 가장 절절한 동기를 가진 이들이 직격탄. 50명 중 과반이 이 지점에서 충격.
2
/jarvis 기대 불일치. 대부분 범용 AI(ChatGPT)를 기대 → 실제는 내 위키 RAG. 카페 사장은 "발주 정리해줘"→"제 위키에 없어요"로 D+0 이탈. 개발자·창업가·전문직은 세션 기억 없음 + 스트리밍 없음에 실망.
3
콜드스타트 빈 상태. 가치는 데이터가 쌓여야 나오는데, 빈 홈·빈 결과가 그 전에 사용자를 내보냄. 시니어는 빈 별자리 그래프를 "고장/오류"로 인식(G-2·G-4·H-3).
4
온보딩 은유 과부하. 마을·세포·조각·별자리가 기능 진입 전 인지부하를 높임. 테크 친숙도 낮을수록(자영업·생산직·농업·시니어) 거의 기능 0%.
5
접근성 절벽 — 40대부터 급락. 다크+저대비(mistGray 3.5:1 WCAG 미달) + 픽셀폰트 + 10px 탭 + 작은 터치타겟. 노안(40s)→돋보기(50s)→백내장·황반변성·녹내장(70s+)에서 사실상 의학적 실격.
6
음성 입력 부재. 70대+ 관절염·손떨림·저시력에서 타이핑이 유일 입력인데 막힘. 90대는 직접 사용 ≈0% → 자녀·보조자 대리입력만 현실적.
7
생년월일 텍스트 입력(YYYY-MM-DD, 피커 없음). 60대+ 단독 가입 성공률 절반 이하. 시니어의 첫 탈락 필터이자 자녀 의존 유발점.
8
위기 분류기 오탐. 청소년 은어("죽겠다")·의료직 맥락("번아웃")이 키워드 기반 감지에 걸려 과잉/오분류. 진솔한 기록을 위축시키는 역효과.

02양 끝의 역설 — 같은 해법을 가리킨다

요청한 "글 읽는 영유아부터"라는 범위 자체가 제품의 경계를 드러내는 테스트가 됐다.

미성년 (6–17): 전원 C10에 차단 — 가입 시 만 18세 이상만(UI canSubmit + 클라 AgeGateError + DB CHECK 3겹). 특히 16–17세는 정체성 탐색·문해력·동기를 모두 갖춘 잠재 코어인데 나이 한 줄에 막힌다. (참고: 한국 정보통신망법은 만 14세 자기동의 — C10의 18세는 법보다 4년 보수적.)
초고령 (80–99): 가입은 되지만 저시력·관절염·인지저하로 직접 사용 불가. 대신 "자녀가 대신 입력"을 강하게 원함 — 6.25 증언, 40년 간호 일기, 산업화 1세대 서사 등 강력한 유산 콘텐츠 보유.

두 끝이 같은 구조를 요구한다 — 보호자/보조자 계정 + 음성 파이프라인. 미성년은 보호자 동의·감독, 초고령은 자녀 대리입력. 하나의 아키텍처가 양쪽을 동시에 연다. (이 발견이 현재 진행 중인 연령 게이트 하향 + 보호자 동의 작업의 출발점.)
