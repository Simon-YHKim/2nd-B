[cowork] 2nd-Brain: Firebase 콘솔에서 Android 앱 등록 + 빌드 시크릿 3종 배선.
브라우저는 Simon 구글 계정 로그인 상태다. 단계별로 어디를 클릭할지 안내하고, 각 단계
완료를 확인한 뒤 다음으로 넘어간다. google-services.json의 내용은 채팅에 출력하지 말고
경로로만 다룬다.

목표 산출물 4종:
 1) E:\2ndB\google-services.json  (실제 파일; 리포에서 gitignored 확인됨)
 2) EAS file env var GOOGLE_SERVICES_JSON  (development / preview / production 전부)
 3) GitHub repo secret GOOGLE_SERVICES_JSON_BASE64
 4) Firebase-GA4 연결 확인 (권장)

절차:
A. https://console.firebase.google.com → "프로젝트 추가"
   - 이름 예: secondbrain-prod (쓸만한 기존 프로젝트가 있으면 재사용 가능)
   - Google 애널리틱스 활성화 선택 → 산출물 4)를 여기서 해결
B. 프로젝트 개요 → Android 아이콘(앱 추가)
   - 패키지 이름: com.simonk.secondbrain   ← 오타 금지. 틀리면 gradle이 패키지
     불일치로 빌드 실패함
   - 앱 닉네임: 2nd-Brain / 디버그 서명 인증서 SHA-1: 지금은 생략 (Analytics만이면 불필요)
C. google-services.json 다운로드 → E:\2ndB\google-services.json 으로 저장
   - 저장 후 터미널 확인: cd E:\2ndB ; git check-ignore google-services.json
     → 경로가 출력되어야 정상(= 커밋 안 됨)
D. EAS file env 등록 (터미널):
   cd E:\2ndB
   eas env:create --scope project --name GOOGLE_SERVICES_JSON --type file --value ./google-services.json --visibility secret
   - 환경을 물으면 development / preview / production 셋 다 선택
   - eas-cli 버전에 따라 플래그가 다르면 eas env:create --help 를 보거나,
     expo.dev 대시보드 → 프로젝트 → Environment variables → Add variable (type: File)로 등록
E. GitHub secret 등록 (PowerShell):
   $b64=[Convert]::ToBase64String([IO.File]::ReadAllBytes("E:\2ndB\google-services.json"))
   gh secret set GOOGLE_SERVICES_JSON_BASE64 --repo Simon-YHKim/2nd-B --body $b64
F. 검증:
   - gh workflow run android-release.yml --repo Simon-YHKim/2nd-B 실행 후 해당 run 로그의
     "Provision google-services.json" 스텝에 "Real google-services.json provisioned from
     secret." 이 찍히는지 (placeholder 경고가 사라져야 함)
   - eas build --profile development --platform android 가 prebuild를 통과하는지
   - 런타임 매트릭스: docs/native-analytics-verification-prompt_260718.html 섹션 B4·B5
     (성인 QA 계정 external_analytics ON에서만 DebugView 이벤트, minor/미확인 0건)

대안: C까지만 끝내고 Claude Code 세션에 "google-services.json 준비됨"이라고 알리면
D·E는 Claude Code가 CLI로 마무리할 수 있다.

보고 형식: 산출물 4종 각각 완료/실패 + 실패 항목은 마지막 화면 상태 요약.
