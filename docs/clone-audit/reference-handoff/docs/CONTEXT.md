# CONTEXT.md — 이 디자인이 만들어진 과정과 배경

Claude(디자인 세션)와 사용자가 여러 차례 반복하며 만든 결과물이다. 대화 컨텍스트를 이 문서로 압축했다.
"왜 이렇게 생겼는가"에 대한 답은 대부분 여기와 PRD에 있다.

## 1. 제작 히스토리 (요약)

1. **기획 문서 업로드** — 사용자가 앱 기획(기능 정의, 별자리 홈 화면설계, 건강 데이터 카탈로그, 취미 입력 UI, 정성 캡처 설계, 손그림 스케치)을 가져옴. 핵심 컨셉인 "별자리 홈 + 세컨비 캐릭터"는 이 원안에서 왔다.
2. **PRD 작성** — 원안을 정리해 `docs/PRD (standalone).html`로 재작성. 레이어 구조(A=삶의 도메인 입력 → B=숨은 검증 레이어(Big Five·애착·가치관·SDT) → C=북극성 종합), 비차단 분석 원칙, 데이터 주권(IDEN·내보내기) 등이 여기서 확정.
3. **프로토타입 구축** — Material 3를 기반으로 하되 M3 번들 컴포넌트가 아닌 **수제 M3 토큰 테마(`m3-theme.css`) + 자체 프리미티브**로 구현(딥스페이스 커스텀이 많아 번들보다 직접 구현이 정확했음). 별자리 홈부터 시작해 화면을 증분 추가, 최종 약 60화면.
4. **반복 개선** — 주요 결정들: 별 7개를 심리 구인이 아닌 **삶의 도메인**으로(심리 검증은 숨은 B레이어로 이동, PRD §4 반영) · 모든 서브 화면을 별하늘 위에 뜨는 "창(window)"으로 통일 · 위키/관계 그래프를 같은 물리 시뮬레이션 시스템으로 통일 · 뮤지엄을 GT7 스타일 2축 타임라인+전시 덱으로 · 컴패니언 말풍선을 트래킹 카피가 아닌 "관찰 문구 순환"으로.
5. **화면 명세서 캡처** — `window.__sb.jump/overlay` 디버그 API로 37개 대표 화면을 390px로 캡처해 `docs/Screen-Spec/` 생성.
6. **핸드오프** — 레거시 반복 산출물(중간 스크린샷 292장, 원본 업로드) 제거 후 이 번들 구성.

## 3. 아키텍처가 이런 이유 (프로토타입 관점)

- **빌드 없는 단일 HTML + Babel jsx 다중 파일**: 디자인 반복 속도 때문. 각 `<script type="text/babel">`은 스코프가 분리되므로 파일 끝에서 `Object.assign(window, {...})` / `window.SB.*`로 공유한다. **로드 순서 = 의존 순서** (2nd-Brain.html의 스크립트 순서 유지 필요).
- **`window.SB` 네임스페이스**: 데이터(C 토큰 헬퍼, STARS, NAV, CHAT_MODES, COMPANION, OBSERVATIONS, RECORDS, BIGFIVE, ERAS, CAPTURE_MODES, WIKI_DOMAINS/RECS, DOMAIN_META, StarGauge, HealthLens)의 단일 출처. 실코드 이식 시 이 데이터 모듈만 옮기면 전 화면 내용이 따라온다.
- **시드 고정 난수**: 배경 별 96개(시드 `70730219`), 그래프 초기 배치 등은 결정적(deterministic)이라 리로드해도 같은 그림. 클론에서도 시드까지 복사해야 "완전 동일"이 성립한다.
- **그래프 물리**: wikigraph/relgraph는 중심장력·반발력·링크장력을 rAF로 시뮬레이션. 파라미터 값은 각 파일 상단 상수 — 그대로 이식.
- **분석은 절대 화면을 막지 않는다** (PRD 불변식): 모달 진행바 금지, AnalysisDock + 토스트만.
- **접근성/기타**: 터치 타깃 44px+, `prefers-reduced-motion` 대응, 스크롤바 숨김, 상태바는 실시간 시각.

## 4. 의도적으로 가짜인 것 (제품 구현 시 실데이터로 대체)

- 챗 응답·분석 잡(랜덤 진행률)·연동 상태·별가루 124개 카운트 등 모든 데이터는 목데이터.
- 로그인 프로바이더 버튼은 UI만. 통화 녹음·건강 연동도 시뮬레이션.
- `image-slot`(뮤지엄 도판 등)은 사용자가 이미지를 드롭하는 프로토타입 장치.

## 5. Tweaks 패널이 말해주는 것

Tweaks는 프로토타입 검토 도구지만 **디자인 결정의 축**을 보여준다: 팔레트(시안=확정 기본/바이올렛=M3 베이스라인 비교용), 다크(기본 ON), 세컨비 머리 크기·표정, 별 레벨, 모션 강도, 홈/챗/담기 variant(모두 A가 확정안), dataState(채움/빈/로딩/오류/오프라인 상태 화면 확인용). 제품에는 다크/팔레트 전환(설정 화면)만 노출한다.
