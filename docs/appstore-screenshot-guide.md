# App Store 스크린샷 촬영 가이드 (iOS)

> 목적: 아이폰 실기가 확보되는 순간 **10분 안에 촬영을 끝내기** 위한 사전 준비 문서.
> 근거: `docs/VISION.md`의 3축(알아가기 · 개인 비서 · 공상-구체화) + `docs/CONCEPT.md`의 별자리 3레이어(도메인 별 - 검증 구인 - 북극성).
> 대상 빌드: TestFlight 0.1.0 (6) 이후. 계정: QA 공용 계정(`.env.test`의 `QA_TEST_EMAIL`).

## 1. 요구 규격 (App Store Connect)

| 항목 | 값 |
|---|---|
| 필수 세트 | **6.5" 디스플레이** |
| 허용 해상도 | 1242x2688 · 2688x1242 · 1284x2778 · 2778x1284 (px, 세로/가로) |
| 장수 | 최대 10장 (**권장 6장**) |
| 노출 | **앞의 3장만 설치 시트에 보인다** - 순서가 곧 우선순위다 |
| 형식 | PNG 또는 JPG, 알파 채널 없음 |

- 기기 화면이 6.5"/6.7"급이 아니면: 실기 캡처 원본을 **1284x2778로 단순 리샘플**해서 올린다(프레임 합성 없이 비율 유지 리사이즈). ASC는 픽셀 치수만 검사한다.
- 6.7"급(1290x2796) 캡처는 1284x2778로 리사이즈하면 왜곡이 사실상 없다.
- 최종 스펙은 ASC 업로드 화면이 정본이다. 업로드 시 다른 크기를 요구하면 그 화면 기준으로 맞춘다.

## 2. 확정 컷 6장 (제품 서사 순서 = 업로드 순서)

앞 3장 = 설치 시트 노출분: 세계관 훅 → 비서 가치 → 종합 출력. 뒤 3장 = 입력 용이성 · 정직성 · 신뢰 장치.

| # | 화면 (라우트) | 이 컷이 증명하는 것 (한 줄) |
|---|---|---|
| 1 | **별자리 홈** (`/`) | 내 기록이 하나의 별자리(나)로 보인다 - 북극성 우위 + 도메인 별들의 서로 다른 밝기 |
| 2 | **세컨비 대화** (`/secondb`) | 내 데이터를 근거로 답하는 개인 비서 - Analytic/Divergent 모드 구분이 화면에 보이게 |
| 3 | **북극성 종합** (`/northstar` · 폴라리스 탭) | 쌓인 데이터가 '나'라는 종합(페르소나)으로 승화된다 - 반영은 내 비준으로만 |
| 4 | **담아내기** (`/capture`) | 일상이 쉽게 들어간다 - 메모/링크/클립/OCR/파일 5모드 선택 UI가 보이는 상태 |
| 5 | **도메인 별 상세** (홈에서 별 탭 → 드릴다운) | 정직한 밝기 - L1~L5 사다리와 근거 기록이 함께 보인다(빛나는 만큼만 안다) |
| 6 | **리즈닝 결과 검토** (`/reasoning` 실행 완료 상태) | AI는 제안만 하고, 반영할 항목은 항상 사용자가 고른다 |

- 언어: **KO 우선** 촬영(한국 스토어 1차). EN 세트는 언어 전환 후 동일 순서로 재촬영(후속).
- 각 컷에서 상태바는 시간만 정상 노출(방해금지 ON, 배터리 충분, 통신사 표기 무관).

## 3. 각 컷의 사전 조건 (빈 화면 금지)

빈 상태 스크린샷은 심사·전환 양쪽에서 최악이다. QA 계정에는 07-18 QA 잔여 데이터(카카오 임포트 소스, 별칭 인물, 일부 비준 태그)가 있으나 아래 수준까지 보강한다.

| # | 컷 | 필요한 데이터 상태 |
|---|---|---|
| 1 | 홈 | 3개 이상 도메인 별이 서로 다른 밝기(비준된 `domain:*` 태그 분산) + 북극성 문장 존재 |
| 2 | 세컨비 | 대화 2~3턴(질문 예: "요즘 내 기록에서 반복되는 흐름이 뭐야?") - 근거 인용이 보이는 답변 + 모드 UI |
| 3 | 북극성 | 북극성 제안 → 비준 1회 완료(페르소나 문장 노출 상태) |
| 4 | 담아내기 | 5모드 셀렉터가 보이는 화면. 메모 모드에 2~3줄 입력 중 상태가 더 좋다 |
| 5 | 별 상세 | 해당 도메인에 기록 5건 이상 + L2 이상(레벨 배지와 근거 리스트가 채워져 보이게) |
| 6 | 리즈닝 검토 | 미비준 자료 3~5건이 제안 리스트로 뜬 '검토' 상태(반영 전) |

### QA 계정 시드 절차 (실기 촬영 전, 15분)

TestFlight 빌드는 프로덕션 env라 `EXPO_PUBLIC_FORCE_TIER`를 쓸 수 없다. **free 티어 한도(리즈닝 주 2회) 안에서** 아래 순서로 시드한다. 시드는 실기 전에 웹(라이브)에서 미리 해도 된다 - 같은 계정이면 실기에 그대로 보인다.

1. `.env.test` 자격으로 로그인.
2. `/capture`에서 기록 12~15건 입력 - 커리어/건강/성장/관계 등 도메인이 갈리도록 소재 분산(운동 메모, 업무 회고, 책 링크, 가족 통화 메모 등).
3. `/reasoning`에서 5건 선택 → 실행(주간 1회차 소모) → **이 '검토' 화면에서 컷 6을 먼저 촬영** → 전체 비준.
4. 비준 직후 홈으로 - 별 밝기가 갱신된 상태에서 **컷 1 촬영**, 밝아진 별을 탭해 **컷 5 촬영**.
5. `/northstar`에서 제안 → 비준 → **컷 3 촬영**.
6. `/secondb`에서 2~3턴 대화 후 **컷 2 촬영**(모드 전환 UI가 프레임에 들어오게).
7. `/capture`로 이동해 메모 2~3줄 입력 중 상태로 **컷 4 촬영**.

> 리즈닝 주간 잔여가 0이면 월요일 00:00 KST 리셋 후 촬영하거나, 촬영 주간에는 시드용 실행을 1회로 아낀다.

## 4. 캡처 → 전송 → 파일명 → 업로드

1. **캡처**: 측면 버튼 + 음량 올리기 동시 누름. 촬영 직후 미리보기에서 잘림/알림 배너 유무 확인(배너가 찍혔으면 재촬영).
2. **전송**: AirDrop(Mac) 또는 iCloud/Google Drive **원본 화질**로 PC 전송. 메신저 전송은 압축되므로 쓰지 않는다(불가피하면 '파일로 첨부').
3. **파일명 규칙**: `ios-6.5_{순번2자리}-{슬러그}_{가로x세로}.png`
   예) `ios-6.5_01-home_1284x2778.png` · `ios-6.5_06-reasoning-review_1284x2778.png`
   순번 = 2절의 컷 번호. 리샘플했다면 리샘플 후 해상도로 표기.
4. **업로드**: App Store Connect → 나의 앱 → 2nd-Brain → App Store 탭 → 준비 중인 iOS 버전 → '미리보기 및 스크린샷' → **6.5" 디스플레이** 탭 → 드래그 업로드 → 1~6 순서 배치 → 저장.

## 5. 하지 말 것

- **기기 프레임 목업 합성 금지**(순수 화면 캡처만).
- **과장 카피 오버레이 금지**(심사 지적 클래스 - 화면 위 텍스트 얹기 없이 화면 그 자체로 증명).
- 임포트 원문·연락처류가 크게 보이는 프레임 금지(별칭 시스템이 가리지만, 원문 노출 컷은 피한다).
- 위기·민감 소재가 보이는 대화 프레임 금지.
- 디버그/QA 전용 UI(judge 배지, dev 표식)가 보이는 컷 금지.
