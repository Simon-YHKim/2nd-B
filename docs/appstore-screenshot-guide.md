## 1. 요구 규격 (App Store Connect)

| 세트 | 허용 해상도 (px) | 어떤 기기 캡처인가 |
|---|---|---|
| **6.9" 디스플레이** | 1290x2796 · 2796x1290 · **1320x2868 · 2868x1320** | iPhone Pro Max급 실기. **캡처 원본을 그대로 업로드한다(리샘플 금지)** |
| **6.5" 디스플레이** | 1242x2688 · 2688x1242 · 1284x2778 · 2778x1284 | 구형 6.5"급 실기 캡처 |

- 촬영 기기의 캡처 해상도가 위 표 중 하나와 일치하면 **그 세트에 원본 그대로** 올린다.
- 어느 표에도 없는 해상도의 기기라면: **지원 기기(또는 Xcode 시뮬레이터)에서 재촬영이 우선**이다.
  임의 비율을 정확 치수로 "비율 유지 리사이즈"하는 것은 일반적으로 불가능하다(치수가 어긋나면 ASC가
  거부하고, 강제로 맞추면 UI가 왜곡된다). 재촬영이 불가할 때만: 목표 비율로 **중앙 크롭 → 정확 치수
  리사이즈 → 육안 검증** 순서로 만들고, 잘린 가장자리에 UI 요소가 없는지 확인한다.
- 장수 최대 10장(권장 6장), **앞의 3장만 설치 시트에 노출** - 순서가 곧 우선순위다. PNG/JPG, 알파 없음.
- 최종 스펙은 ASC 업로드 화면이 정본이다. 요구 세트가 다르면 그 화면 기준으로 맞춘다.

## 2. 확정 컷 6장 (제품 서사 순서 = 업로드 순서)

앞 3장 = 설치 시트 노출분: 세계관 훅 → 비서 가치 → 종합 출력. 뒤 3장 = 입력 용이성 · 정직성 · 신뢰 장치.

| # | 화면 (실제 진입 경로) | 이 컷이 증명하는 것 (한 줄) |
|---|---|---|
| 1 | **별자리 홈** (`/`) | 내 기록이 하나의 별자리(나)로 보인다 - 북극성 우위 + 도메인 별들의 서로 다른 밝기 |
| 2 | **세컨비 대화** (`/secondb`) | 내 데이터를 근거로 답하는 개인 비서 - 세 페르소나(2nd-B·메타비·트위비) 선택 UI가 보이게 |
| 3 | **북극성 종합** (홈에서 북극성 탭 → `/core-brain`, PolarisDeck) | 흩어진 기록·응답이 근거와 함께 '나'라는 종합(페르소나·강점·자기초상)으로 정리된다 |
| 4 | **담아내기** (`/capture`, 모드 확장 상태) | 일상이 어떤 형태로든 들어간다 - 글/메모/4W1H/링크·클립/사진(OCR)/음성/할 일/파일 8모드 |
| 5 | **도메인 별 상세** (홈에서 별 탭 → `/star/{domain}`) | 정직한 밝기 - L1~L5 사다리와 근거 기록 리스트가 함께 보인다 |
| 6 | **리즈닝 결과 검토** (`/reasoning` 실행 완료 상태) | AI는 제안만 하고, 반영할 항목은 항상 사용자가 고른다 |

- 컷 3 주의: `/northstar`는 한 줄 문장 제안·편집 화면이다. **종합 덱은 홈의 북극성을 탭해 열리는 `/core-brain`**(PolarisDeck)이다.
  이 덱의 페르소나 종합은 마운트 시 `buildPersona()`가 자동 생성한다(`src/app/core-brain.tsx`) - **비준 대상이 아니므로
  "반영은 비준으로만"류 주장은 이 컷에 쓰지 않는다.** 그 메시지는 실제 비준 표면인 컷 6 전용이다.
- 컷 4 주의: 기본 노출 모드는 일부이고 **확장(더 보기) 후 8모드**가 전부 보인다(`CAPTURE_MODES`: journal·memo·fourw·linkclip·ocr·voice·todo·file - 링크와 클립은 결합 모드). 확장 상태로 촬영한다.
- 컷 2 주의: 캐노니컬 `/secondb`(파라미터 없는 진입)는 **세 페르소나 셀렉터**를 렌더한다(`REV2_PERSONA_IDS`; Analytic/Divergent 토글 UI는 `?character=` 딥링크의 레거시 캐릭터 분기 전용 - `src/app/secondb.tsx:470`). 분석·발산의 서사는 메타비(분석 결)·트위비(발산 결) 페르소나 구분으로 전달한다.
- 컷 5 주의: **타깃 도메인은 record 리스트를 렌더하는 렌즈로 고른다(커리어·성장·담아내기 중 택1, 예시는 커리어).**
  건강은 `DomainStarLens`가 `HealthLens`(health_samples 구조 데이터)로 분기해 **record 리스트를 그리지 않는다** - 5건을 비준해도 빈 패널이 찍힌다.
- 언어: **KO 우선** 촬영. 상태바는 시간만 정상 노출(방해금지 ON, 배터리 충분).

## 3. 각 컷의 사전 조건 (빈 화면 금지)

| # | 컷 | 필요한 데이터 상태 |
|---|---|---|
| 1 | 홈 | 타깃 도메인 별 1개가 L3, 다른 별 1~2개가 L2로 **밝기가 실제로 갈린 상태** (북극성 별은 항상 렌더된다 - 별도 문장 시드 불요, 아래 주의 4) |
| 2 | 세컨비 | **촬영 기기에서 직접** 만든 2~3턴 - **시드한 기록 주제로 질문**해 근거 인용이 뜨는 프레임(아래 주의 5) |
| 3 | 북극성 종합 | records + **강점 검사(`/strengths`) 완료**(덱의 강점 칩 재료; 미완이면 "강점 체크를 마치면…" 빈 상태가 찍힌다 - 아래 주의 6) |
| 4 | 담아내기 | 모드 확장 상태. 메모 모드에 2~3줄 입력 중이면 더 좋다 |
| 5 | 별 상세 | **타깃 도메인(커리어 권장)에 records 5건 이상 비준**(임포트 소스 금지 + 건강 도메인 금지 - 위 컷 5 주의) |
| 6 | 리즈닝 검토 | 타깃 도메인 records 5~6건이 제안 리스트로 뜬 '검토' 상태 - **웹에서 실행만 하고 비준하지 않은 채** 기기에서 복원(아래 주의 2) |

### 소스 재검증에서 나온 주의 7가지

1. **자동 리즈닝을 먼저 끈다.** QA 계정은 서버 저장 자동 토글이 ON 상태다(0093, 07-18 QA 잔여). 켠 채로 시드하면 `/capture` 저장마다 `enqueueAutoReasoningRecord`가 주간 1회를 소모하고 1건짜리 제안이 검토 화면을 선점한다. 시드 전 설정에서 OFF + `/reasoning`의 잔존 제안을 정리한다.
2. **컷 6은 기기에서 찍는다 - 웹에서 비준하지 말 것.** 제안은 서버 영속이라(`src/app/reasoning.tsx` 0092 복원) 웹에서 실행만 해두면 기기 `/reasoning`이 검토 상태를 그대로 복원한다. 웹에서 비준해 버리면 검토 화면이 사라져 컷 6을 잃는다.
3. **세컨비 대화는 웹 사전 시드가 불가능하다.** 턴은 컴포넌트 로컬 상태, 서버엔 일일 카운터만 영속(`src/app/secondb.tsx`). 웹 턴은 공유 일일 5턴을 깎기만 한다 - 촬영일 웹 대화 금지, 컷 2는 기기에서 만든다.
4. **북극성 '문장'은 컷 1·3 어디에도 렌더되지 않는다.** 저장 문장을 읽는 화면은 `/northstar`와 `/share-card`뿐(`fetchCurrentNorthstar` 사용처 전수). 홈은 별 밝기만(`DeepSpaceShell`이 starLevels·northStarBrightness 전달), `/core-brain`은 문장을 읽지 않는다. 따라서 **북극성 제안/저장 단계는 촬영 전제에서 제외** - 리즈닝 소모는 총 1회면 된다.
5. **세컨비 인용은 위키·기록 컨텍스트에서 나온다**(`sendChatMessage` → `retrieveChatContext`+`exportUserWiki`). 시드와 무관한 질문이면 인용 칩 없는 일반 답변이 찍힐 수 있다. 시드한 기록 주제(예: "요즘 커리어 기록에서 반복되는 흐름이 뭐야?")로 묻고, 인용이 안 뜨면 질문을 시드 주제로 좁혀 재시도한다.
6. **컷 3의 덱을 채우는 건 검사 데이터다.** 페르소나 채점 입력은 `audit_response`(+TIPI)인데(`build.ts` v1 헤더), 캐노니컬 `/audit`는 `/interview`로 열리고 그 저장 행은 `interview` 태그(질문 프롬프트 없음)라 **채점에 들어가지 않는다**(`src/app/interview.tsx`). 강점 칩은 별도 강점 검사에서 온다 - 미완이면 덱이 "강점 체크를 마치면…" 빈 상태를 그린다(`core-brain.tsx:486-496`). 따라서 시드는 **`/strengths` 강점 검사 완료(필수) + `/big-five` 1회(권장)**로 하고, 컷 3이 증명하는 건 강점·자기초상·근거가 채워진 종합 덱이다.
7. **밝기 밴드는 결정론적이다**: 0건=L1 · 1~4건=L2 · 5~14건=L3(`star-levels.ts`). 그리고 별 상세 근거 리스트는 records 테이블만 읽는다(`star/[domain].tsx`). **타깃 도메인 records를 정확히 5~6건 선택·비준**해야 타깃만 L3 + 리스트가 채워진다(임포트 소스·타 도메인 위주 선택 금지).

### 시드 절차 (웹 선작업 + 기기 당일, 총 리즈닝 소모 1회)

records 입력·리즈닝 실행·비준·audit 응답은 서버 영속이라 **웹 선작업이 아이폰에 그대로 보인다**(대화와 아래 D0의 기기 로컬 플래그만 예외). TestFlight 빌드는 프로덕션 env라 `EXPO_PUBLIC_FORCE_TIER` 사용 불가 - free 한도 안에서 설계됐다.

**웹 (촬영 전날까지):**

- W0. **사전 점검(preflight)**: 공유 QA 계정에는 기존 데이터가 있다 - `loadDomainLevels()`는 이번 시드만이 아니라 **계정의 전체 태그 행+구조 테이블(관계·휴식·재정·성장)**을 센다. 홈에서 현재 별 레벨을 먼저 보고, ① 타깃은 "지금 L2 이하(총 15건 미만)"인 record-lens 도메인으로 고르고 ② 비교 별로 남길 도메인에 이미 5건 이상 있으면 다른 도메인을 비교용으로 삼는다. 목표 상태 = 타깃 총 5~14건(L3), 비교 별 총 1~4건(L2).
- W1. 설정에서 **자동 리즈닝 OFF** + `/reasoning`에 잔존 제안이 있으면 정리(비준 또는 기각). 그 다음 주간 잔여 **1회 이상** 확인.
- W2. `/capture`: **타깃 도메인 소재 records를 W0 계산에 맞춰 5~6건** + 비교 도메인 2곳 1~2건씩. 임포트는 쓰지 않는다.
- W3. `/strengths` 강점 검사 완료(필수) + `/big-five` 1회(권장) - 컷 3 덱 재료(주의 6).
- W4. `/reasoning`: **타깃 도메인 records를 정확히 5건 선택**(선택 상한 `MAX_SELECTION = 5` - 6번째는 조용히 선택되지 않는다, `reasoning.tsx:59`) → 실행 [주간 1회 소모] → 제안 확인 후 **비준하지 않고 그대로 둔다**(검토 상태 서버 보존).

**기기 (촬영 당일, TestFlight 설치 직후):**

- D0. **워밍업**: QA 계정 로그인 → 온보딩 완료 → TTFV 흐름 통과 → 홈 코치마크 4단계 해제. 이 셋은 **기기 로컬 플래그**(AsyncStorage)라 웹 선작업이 안 먹는다 - 건너뛰면 촬영이 리다이렉트/오버레이 뒤에서 시작된다.
- D1. `/reasoning` → 웹에서 남겨둔 **검토 상태가 복원**된 화면에서 **컷 6 촬영** → 기기에서 전체 비준.
- D2. **홈을 재진입해서**(리즈닝에서 Back으로 돌아가지 말 것 - `DeepSpaceShell`은 마운트/유저 변경 시에만 레벨을 다시 읽어 Back 복귀 화면은 비준 전 밝기를 계속 보여줄 수 있다, `DeepSpaceShell.tsx:47-67`) **컷 1 촬영**(타깃 별 L3 vs 비교 별 L2) → 타깃 별 탭 → **컷 5 촬영**(records 근거 리스트).
- D3. 홈에서 북극성 탭 → `/core-brain` 덱에서 **컷 3 촬영**(강점 칩·자기초상·근거가 채워진 종합).
- D4. `/secondb`: 시드 주제로 2~3턴(일일 5턴 내) → 인용 칩 + **페르소나 셀렉터(2nd-B·메타비·트위비)**가 프레임에 들어오게 **컷 2 촬영**.
- D5. `/capture` 모드 확장 상태로 **컷 4 촬영**.

## 4. 캡처 → 전송 → 파일명 → 업로드

1. **캡처**: 측면 버튼 + 음량 올리기. 직후 미리보기에서 잘림/알림 배너 확인(배너가 찍혔으면 재촬영).
2. **전송**: AirDrop(Mac) 또는 iCloud/Google Drive **원본 화질**. 메신저 전송은 압축되므로 금지(불가피하면 '파일로 첨부').
3. **파일명 규칙**: `ios-{세트}_{순번2자리}-{슬러그}_{실해상도}.png`
   예) `ios-6.9_01-home_1320x2868.png` · `ios-6.5_06-reasoning-review_1284x2778.png`
   세트 = 캡처가 속한 디스플레이 세트(6.9 또는 6.5). 순번 = 2절의 컷 번호.
4. **업로드**: App Store Connect → 나의 앱 → 2nd-Brain → App Store 탭 → 준비 중인 iOS 버전 → '미리보기 및 스크린샷' → **캡처 해상도에 맞는 세트 탭** → 드래그 업로드 → 1~6 순서 배치 → 저장.
