## 2. 확정 컷 6장 (제품 서사 순서 = 업로드 순서)

앞 3장 = 설치 시트 노출분: 세계관 훅 → 비서 가치 → 종합 출력. 뒤 3장 = 입력 용이성 · 정직성 · 신뢰 장치.

| # | 화면 (실제 진입 경로) | 이 컷이 증명하는 것 (한 줄) |
|---|---|---|
| 1 | **별자리 홈** (`/`) | 내 기록이 하나의 별자리(나)로 보인다 - 북극성 우위 + 도메인 별들의 서로 다른 밝기 |
| 2 | **세컨비 대화** (`/secondb`) | 내 데이터를 근거로 답하는 개인 비서 - Analytic/Divergent 모드 구분이 화면에 보이게 |
| 3 | **북극성 종합** (홈에서 북극성 탭 → `/core-brain`, PolarisDeck) | 쌓인 데이터가 '나'라는 종합으로 승화된다 - 반영은 내 비준으로만 |
| 4 | **담아내기** (`/capture`, 모드 확장 상태) | 일상이 어떤 형태로든 들어간다 - 글/메모/4W1H/링크·클립/사진(OCR)/음성/할 일/파일 8모드 |
| 5 | **도메인 별 상세** (홈에서 별 탭 → `/star/{domain}`) | 정직한 밝기 - L1~L5 사다리와 근거 기록 리스트가 함께 보인다 |
| 6 | **리즈닝 결과 검토** (`/reasoning` 실행 완료 상태) | AI는 제안만 하고, 반영할 항목은 항상 사용자가 고른다 |

- 컷 3 주의: `/northstar`는 한 줄 문장 제안·편집 화면이다. **종합 덱은 홈의 북극성을 탭해 열리는 `/core-brain`**(PolarisDeck)이다.
- 컷 4 주의: 기본 노출 모드는 일부이고 **확장(더 보기) 후 8모드**가 전부 보인다(`CAPTURE_MODES`: journal·memo·fourw·linkclip·ocr·voice·todo·file - 링크와 클립은 결합 모드). 확장 상태로 촬영한다.
- 언어: **KO 우선** 촬영. 상태바는 시간만 정상 노출(방해금지 ON, 배터리 충분).

## 3. 각 컷의 사전 조건 (빈 화면 금지)

| # | 컷 | 필요한 데이터 상태 |
|---|---|---|
| 1 | 홈 | 타깃 도메인 별 1개가 L3, 다른 별 1~2개가 L2로 **밝기가 실제로 갈린 상태** + 북극성 문장 저장됨 |
| 2 | 세컨비 | **촬영 기기에서 직접** 만든 2~3턴 대화(대화는 기기 로컬 상태 - 아래 주의) |
| 3 | 북극성 종합 | records 축적 + 북극성 문장 존재(덱 페이지가 채워져 보이게) |
| 4 | 담아내기 | 모드 확장 상태. 메모 모드에 2~3줄 입력 중이면 더 좋다 |
| 5 | 별 상세 | 타깃 도메인에 **records 기반** 기록 5건 이상 비준(임포트 소스는 이 화면 근거 리스트에 안 뜬다 - 아래 주의) |
| 6 | 리즈닝 검토 | 미비준 records 5건이 제안 리스트로 뜬 '검토' 상태(반영 전) |

### 소스 재검증에서 나온 주의 3가지

1. **세컨비 대화는 웹 사전 시드가 불가능하다.** 대화 턴은 컴포넌트 로컬 상태이고 서버에는 일일 사용 카운터만 영속된다(`src/app/secondb.tsx` 헤더 주석). 같은 계정으로 로그인해도 웹에서 만든 대화는 아이폰에 나타나지 않는다. 게다가 카운터는 계정 공유라 **웹에서 턴을 쓰면 기기 몫이 줄어든다**(free 일일 5턴). 촬영일에는 웹에서 대화 턴을 소비하지 말고, 컷 2는 기기에서 2~3턴으로 직접 만든다.
2. **밝기 밴드는 결정론적이다**: 기록 0건=L1 · 1~4건=L2 · 5~14건=L3(`src/lib/persona/star-levels.ts`). 12~15건을 4개 도메인에 고르게 뿌리면 전부 L2가 되어 "별마다 밝기가 다르다"(컷 1)와 "5건 이상"(컷 5)이 동시에 깨진다. **타깃 도메인 1곳에 5~6건, 다른 도메인 2곳에 1~2건씩**으로 시드한다.
3. **별 상세의 근거 리스트는 records 테이블만 읽는다**(`src/app/star/[domain].tsx`). 리즈닝 선택 목록에는 records와 임포트 소스가 섞여 있고 QA 계정엔 카카오 임포트 소스가 남아 있다 - 소스를 비준하면 별은 밝아지지만 컷 5의 리스트는 비어 보인다. 컷 5용 비준은 **records만 골라서** 한다.

### QA 계정 시드 절차 (실기 촬영 전, 15분)

TestFlight 빌드는 프로덕션 env라 `EXPO_PUBLIC_FORCE_TIER`를 쓸 수 없다. records 입력·비준·북극성 저장은 서버 영속이라 **웹에서 미리 해도 아이폰에 그대로 보인다**(대화만 예외). 리즈닝 소모는 아래 순서 기준 A안 2회 / B안 1회.

0. **잔여 확인**: 주간 리즈닝 잔여가 A안(북극성 제안)이면 **2회 이상**, B안(북극성 수동 입력)이면 1회 이상인지 확인한다. `/northstar`의 제안도 성공 시 리즈닝 사용량을 1회 소모한다(`src/app/northstar.tsx`의 `incrementReasoningUsage`). 부족하면 월요일 00:00 KST 리셋 후 진행.
1. `.env.test` 자격으로 로그인.
2. `/capture`: **타깃 도메인(예: 건강) 소재로 records 5~6건** + 다른 도메인 2곳(예: 커리어·성장)에 1~2건씩, 총 8~10건. 임포트는 쓰지 않는다.
3. `/reasoning`: **records만 5건 선택**(임포트 소스 제외) → 실행 [1회차 소모] → 이 '검토' 화면에서 **컷 6 촬영** → 전체 비준.
4. 북극성 문장 확보 - 둘 중 하나:
   - A안: `/northstar`에서 제안 [2회차 소모] → 마음에 드는 문장 비준·저장.
   - B안: `/northstar`에서 문장을 직접 입력·저장(리즈닝 소모 0).
5. 홈에서 **컷 1 촬영**(타깃 별 L3 vs 나머지 L2 + 북극성) → 타깃 별 탭 → **컷 5 촬영**(records 근거 리스트).
6. 홈에서 북극성 탭 → `/core-brain` 덱에서 **컷 3 촬영**.
7. **[기기에서]** `/secondb` 2~3턴 대화(일일 5턴 한도 내) → 모드 전환 UI가 프레임에 들어오게 **컷 2 촬영**.
8. `/capture` 모드 확장 상태로 **컷 4 촬영**.

## 4. 캡처 → 전송 → 파일명 → 업로드

1. **캡처**: 측면 버튼 + 음량 올리기. 직후 미리보기에서 잘림/알림 배너 확인(배너가 찍혔으면 재촬영).
2. **전송**: AirDrop(Mac) 또는 iCloud/Google Drive **원본 화질**. 메신저 전송은 압축되므로 금지(불가피하면 '파일로 첨부').
3. **파일명 규칙**: `ios-{세트}_{순번2자리}-{슬러그}_{실해상도}.png`
   예) `ios-6.9_01-home_1290x2796.png` · `ios-6.5_06-reasoning-review_1284x2778.png`
   세트 = 캡처가 속한 디스플레이 세트(6.9 또는 6.5). 순번 = 2절의 컷 번호.
4. **업로드**: App Store Connect → 나의 앱 → 2nd-Brain → App Store 탭 → 준비 중인 iOS 버전 → '미리보기 및 스크린샷' → **캡처 해상도에 맞는 세트 탭** → 드래그 업로드 → 1~6 순서 배치 → 저장.

## 5. 하지 말 것

- **기기 프레임 목업 합성 금지**(순수 화면 캡처만).
- **과장 카피 오버레이 금지**(심사 지적 클래스 - 화면 위 텍스트 얹기 없이 화면 그 자체로 증명).

- 위기·민감 소재가 보이는 대화 프레임 금지.
- 디버그/QA 전용 UI(judge 배지, dev 표식)가 보이는 컷 금지.
