> VISION 축 (2) **개인 비서 기반**. 비서 AI의 *목적·역할·실효성·다른 앱과의 연동성*을
> 코드 근거로 점검하고, 합리적 다음 수를 우선순위화한다. 동반 문서:
> `docs/ASSISTANT_OPS.md`(설계), `docs/PERSONAL-ASSISTANT-ROADMAP.md`(14영역 소스 매핑).
>
> 검증 방법: (A) 연동 표면 코드 감사 + (B) SimonK-stack `persona-simulation` 스킬
> 방법론으로 비서 코어 루프를 실제 화면 코드 위로 강제 완주(코호트 12명, file:line 근거).

## 1. 비서 AI의 목적·역할 (재확인)

- **목적**: 사용자 자신의 데이터(기록·위키·평가·완료이력) + 연동 앱 데이터를 근거로
  AI가 작은 루틴을 *추천*하고, 추천에 머물지 않고 **사용자의 캘린더·투두·알림 앱으로
  밀어 넣어** 실제로 굴러가게 한다. "추천에서 끝나면 비서가 아니다."
- **역할 경계**: 자동 실행 없음 — 사용자가 화면에서 누른 항목만 나간다("내가 켠 항목만"
  서약). 의료·임상 아님(어휘 정책). AI는 최후수단, 항상 C1 게이트웨이 경유(harness-first).
- **3계열 구조**: `[데이터 in] → [C1 추천 엔진] → [실행 out: 캘린더/투두/알림]`.

## 2. 현 상태 진단 (2축 감사)

### 2-1. 연동성(OUT) — 능력은 성숙, 정본 화면이 과소 배선
| 단계 | 상태 | 근거 |
|---|---|---|
| P1 ICS(.ics)·Google Cal URL·공유시트 텍스트 | ✅ 구현·테스트 | `src/lib/ops/push.ts:55–108`, `__tests__/push.test.ts` |
| P2 기기 캘린더(expo-calendar)·로컬 알림(expo-notifications) | ✅ 코드 완성, 네이티브 게이트 | `device-calendar.ts`, `reminders.ts` |
| P3 Google Calendar/Tasks API·Health Connect/HealthKit | ❌ 미구현 (OAuth/법무 게이트 G2/G3) | 계획만 |

**핵심 갭**: 레거시 `src/app/ops.tsx`는 5개 푸시 버튼(기기캘린더/Google/.ics/공유/알림)을
다 노출하는데, **정본 deep-space 화면(`DeepSpaceDesignScreens.tsx` Ops)은 Google Cal URL +
공유만** 배선했다. 이미 존재·테스트된 `addEventToDeviceCalendar`(기기 기본 캘린더 →
삼성/애플/Outlook 자동 수렴)를 안 쓴다. → 페르소나-sim P0 #1 "Google 종속/사파리 사인인"의
실제 원인. **새 코드가 아니라 기존 능력의 배선 문제** = $0, 게이트 없음.

### 2-2. 실효성 — 추천이 *실제 행동*에 근거하지 않았다
- 기존 `recommendForDomain`은 **위키 스냅샷(≤600자)만** 근거(`recommend.ts:138`).
  완료 이력·연속일수 등 사용자의 *진짜 행동 신호*를 안 봐서 추천이 매번 일반론에 머묾
  → "비서가 매일 쓰이지 않는" 근본 원인.
- 관리 레이어(저장·알람·완료추적·자동완료[health/focus/SRS])는 이미 14영역 전부 출하됨
  (`routines.ts`) — **신호는 이미 DB에 쌓이는데 추천이 그걸 안 읽었다.**

## 3. 이번 세션 적용 (코드 랜딩 — 화면 무수정)

**적응형 추천 신호(deterministic adherence signal)** 를 추천 엔진에 주입.
- `src/lib/ops/signals.ts` (신규): 사용자 자신의 `ops_routines` + `ops_routine_logs`를
  **집계만** 한 한 줄 요약 — `Recent adherence (last 7d): 5/7 days done; current streak 3d;
  2 active routine(s).` 제목·체크리스트·일기 텍스트 없음(인젝션 표면 0). LLM·새 테이블·새
  의존성 0, $0. 읽기 실패 시 ""로 graceful degrade(추천을 절대 막지 않음).
- `recommend.ts`: 위 신호를 **신뢰(TRUSTED) 사실 줄**로 주입(써드파티 텍스트 아님 →
  `<UNTRUSTED>` 밖). 시스템 프롬프트에 적응 규칙 추가(EN+KO, C7 유지): *꾸준하면 작은 도전,
  흐트러졌으면 더 쉬운 버전, 비난 금지.* 화면 호출부 무수정 → 추천 품질만 상승.
- 테스트 `__tests__/signals.test.ts` 7케이스. `npm run verify` green (202 suites/1608 tests).

> 효과: 같은 영역을 5일 연속 한 사용자에겐 "한 단계 늘려보기", 3일 빵꾸난 사용자에겐
> "5분짜리로 다시 시작"을 제안 — 추천이 살아있게 느껴지는 최소 단위의 실효성 업그레이드.

### DP-1 — 푸시 대상 선택 바텀시트 ("내 앱으로 보내기")
```
2nd-Brain(딥스페이스 다크 코스믹) 비서 화면용 "푸시 대상 선택" 바텀시트를 디자인해줘.

맥락: 사용자가 AI 추천 루틴 카드에서 "내 앱으로 보내기"를 누르면, 어느 앱/형식으로
내보낼지 고르는 바텀시트가 올라온다. 목표는 Google 캘린더 단일 종속을 없애고
플랫폼에 맞는 선택지를 평등하게 제시하는 것(iOS/Outlook/프라이버시 사용자도 막히지 않게).

선택지(상황에 따라 노출/숨김):
- 기기 캘린더에 추가 (네이티브에서만, 기본·1순위 — 삼성/애플/Outlook 자동 수렴)
- Google 캘린더로 열기 (URL 딥링크, 항상 가용)
- 캘린더 파일(.ics) 내려받기/공유 (모든 플랫폼)
- 체크리스트 공유 (공유시트 텍스트, 투두 앱 수신)

요구사항:
- 한 메시지: "이 루틴을 어디로 보낼까요?" + 항목 리스트(아이콘 + 한 줄 라벨).
  설명문 최소화 — 아이콘이 설명이 되게.
- 각 항목 ≥44px 높이, 명확한 탭 피드백. 1순위(기기 캘린더)는 시각적으로 약간 강조(mint),
  나머지는 보조(cyan ghost).
- 최초 1회 동의 카드(기존 ops_push standing-consent)와의 관계: 이미 동의했으면 바로 시트,
  미동의면 시트 상단에 한 줄 동의 + "허용하고 계속".
- 빈/에러 상태: 네이티브 미지원(웹)일 때 "기기 캘린더" 항목은 숨김(빈 dead-end 금지).
- deepSpace 토큰만, hex 0, 바텀시트(노드 위 모달 금지). KO/EN 카피 둘 다 제시.
산출물: 바텀시트 레이아웃 + 항목 컴포넌트 스펙 + 동의 머리글 변형 + 카피(EN/KO).
```

### DP-2 — "예약된 리마인더" 가시화 표면
```
2nd-Brain(딥스페이스) 비서 화면용 "예약된 리마인더" 가시화 표면을 디자인해줘.

문제: 사용자가 루틴을 저장하면 로컬 알림이 예약되는데, 권한 미허용/네이티브 미빌드면
알림이 조용히 안 울려서 "저장됐다는데 안 알려줬다"는 불신이 생긴다. 사용자가 *무엇이
언제 예약됐는지*를 한눈에 보고 끌 수 있어야 한다.

요구사항:
- 한 메시지: "예약된 리마인더" + 항목 리스트(루틴명 · 주기/시간 · 상태칩).
- 상태칩 3종: 활성(mint) / 권한필요(경고색 + "권한 켜기" 인라인 액션) /
  이 기기에서 불가(중립색 + "정식 앱 필요" 안내). 상태가 거짓 성공을 만들지 않게.
- 각 항목에서 끄기(스와이프 또는 명시 토글), ≥44px 타깃.
- 빈 상태: "예약된 리마인더가 없어요" + 루틴 저장 유도 한 줄.
- 진입: Ops 화면 내 섹션 또는 저장 직후 토스트의 "예약 보기" 링크 중 추천안 제시.
- deepSpace 토큰만, hex 0, 정보밀도 규칙. 카피 EN/KO.
산출물: 섹션/리스트 레이아웃 + 상태칩 3종 스펙 + 진입 동선 추천 + 카피(EN/KO).
```

### DP-3 — 추천 카드 접근성·액션 명료화 리스펙
```
2nd-Brain(딥스페이스) Ops 추천 카드의 접근성·액션 명료화 리스펙을 디자인해줘.

현재 문제(페르소나-sim 근거): 본문 폰트 11.5px(노안 안 보임), 버튼 터치타깃 <44px,
Share와 Add-to-calendar 버튼이 시각적으로 똑같아 오탭, 완료 시 피드백 없음.

요구사항:
- 본문(이유) ≥14px, 칩/메타 ≥12px, OS 폰트 스케일 존중(고정 px 지양).
- 모든 액션 버튼 ≥44px(+hitSlop). 1차 액션(루틴으로 저장 또는 보내기)은 mint 강조,
  보조(공유)는 ghost — 색·아이콘으로 구분(공유=내보내기 아이콘, 캘린더=캘린더 아이콘).
- 완료 탭 피드백: 햅틱 + 짧은 스케일 애니메이션 + 연속일수 미세 강조(과한 컨페티 금지,
  딥스페이스 절제 유지).
- 의료조언 아님 고지: 추천 로드 전 1줄, 읽히는 대비/크기로(현재는 로드 후 흐릿).
- 한 화면 = 한 메시지 유지(액션은 카드당 1차 1개 + 보조 최소).
- deepSpace 토큰만, hex 0. 카피 EN/KO.
산출물: 카드 타이포/스페이싱 스펙 + 액션 버튼 2종(1차/보조) 스펙 + 완료 피드백 모션 +
고지 배치 + 카피(EN/KO).
```
