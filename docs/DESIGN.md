## Core Concept — 내 머릿속의 열일하는 세포들

앱의 모든 동작은 사용자의 **'두 번째 뇌'를 짓는 과정**이다.
복잡한 데이터 처리(RAG, 임베딩, 파싱 등)는 만화 **'일하는 세포'** 처럼,
작고 부지런한 세포들이 사용자를 위해 땀 흘리며 뇌를 짓고,
정보를 뛰어다니며 찾아오는 모습으로 비유한다.

기술 용어는 **세포들의 상황**으로 치환해 보여준다.

## Tone & Manner

| 원칙 | 적용 |
|---|---|
| **생동감 넘치는 묘사** | 의성어·의태어 적극 사용 (영차영차, 찌릿, 꾹꾹, 쓱싹, 두근두근…) |
| **기술 용어 금지** | 에러·프로세스 설명을 "세포들의 상황"으로 치환 |
| **다정하고 든든한 태도** | 사용자의 입력을 칭찬, 세포들이 데이터를 얼마나 소중하게 다루는지 보여줌 |
| **경쾌·짧은 단문** | 장황한 설명 금지 — 직관적인 한두 문장 |

## Metaphor Dictionary (용어 치환 사전)

코드와 카피 전부에서 **일관되게** 이 매핑을 따른다.

| 기술 개념 | 세포들의 상황 |
|---|---|
| 데이터 업로드 (Data Ingestion) | 단백질 블록·자재 반입, 세포들에게 식량 주기 |
| 텍스트 분할·임베딩 (Chunking & Embedding) | 뉴런 쪼개기, 뇌 주름 꾹꾹 접기, 정보에 라벨 붙이기 |
| 벡터 검색 (RAG Retrieval) | 일꾼 세포가 기억 저장소로 호다닥 뛰어가서 찾아오기 |
| 답변 생성 (Generation) | 대뇌 통제실 점등, 안테나 수신 완료 |
| 에러 (Error) | 통신 케이블 엉킴, 일꾼 세포가 길 잃음, 시냅스 과부하 |
| 로딩·초기화 (Loading) | Soma → Neuron → Cortex → Cerebrum 단계로 뇌가 자라남 |

## Scenario Examples

이 톤과 어휘를 사용한 응답 예시:

- **파일 업로드 성공**:
  *"자재 반입 완료! 일꾼 세포들이 신나게 뇌 주름을 만들러 달려갔어요."*
- **검색·질문 진행 중**:
  *"기억 저장소로 세포들 출동! 서랍을 열심히 뒤지는 중입니다."*
- **에러 발생**:
  *"앗! 일꾼 세포들이 정보 고속도로에서 길을 잃었어요. 다시 한 번 문서를 확인해 주시겠어요?"*

## Output Rule

모든 문구는 위 가이드를 바탕으로 작성한다.

- ✅ 경쾌·직관적 단문
- ✅ 의성어·의태어 자연스럽게
- ✅ 사용자를 격려하는 어조
- ❌ "임베딩이 완료되었습니다" 같은 기술 어휘 노출
- ❌ "오류 코드 500 발생" 같은 시스템 메시지 그대로 표시

## 점진 마이그레이션 대상

이전 톤(중립·전문가풍)으로 작성된 화면을 이 컨셉에 맞게 단계적으로 갱신:

| 우선순위 | 화면 | 갱신 포인트 |
|---|---|---|
| High | `src/app/capture.tsx` | "캡처" → "자재 반입" 비유로 alert/helper |
| High | `src/app/inbox.tsx` | "Generate wiki page" → "뇌 주름 빚기" |
| High | `src/app/jarvis.tsx` | "Searching wiki…" → "일꾼 세포가 기억 저장소로 출동" |
| Medium | `src/app/wiki.tsx` | export/graph stats |
| Medium | `src/app/insights.tsx` | 통계 문구 톤 다듬기 |
| Low | `src/app/audit.tsx`, `interview.tsx`, `big-five.tsx`, `attachment.tsx`, `mbti.tsx` | 질문은 그대로, 안내·완료 메시지만 갱신 |
| Low | i18n 키 (`locales/ko/*.json`) | 기술 어휘 점검 |

이 컨셉은 **CLAUDE.md "디자인 작업 워크플로"** 와 함께 강제력을 가진다.
새 화면·새 카피를 만들 때 반드시 위 사전을 참고할 것.
