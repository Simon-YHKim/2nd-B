A. 설정 · 자동 리즈닝

목적: 자동 실행 여부와 희소한 주간 횟수를 한눈에 예측하게 한다. 핵심 메시지 1개와 리즈닝 궤도 아이콘 1개만 사용한다.

레이아웃 구조

  1. 공통 windowed deep-space shell과 설정 top app bar
  2. “자동 리즈닝” 단일 설정 행: 궤도 아이콘, 제목, 상태 설명, switch
  3. 같은 카드 안의 잔여 행: 주간 기본 횟수, 보상 크레딧, 초기화 시점
  4. 하단 텍스트 버튼 “자료를 골라 직접 실행”

상태 전수

  • 기본·OFF: 자동 실행 없음. 수동용 횟수 전체 보존.
  • ON: 다음 새 자료부터 자동 묶음 실행. 수동 1회 예약 규칙 표시.
  • 잔여 0: ON 상태는 “일시 정지”로 보이되 switch는 OFF로 변경 가능. 월요일 자동 재개.
  • 실행 중: 행 아래 작은 진행 상태만 표시. 전역 작업 표시와 중복 설명 금지.
  • 완료: “검토할 제안 n개” 텍스트 버튼 노출.
  • 오류: 자동 실행 보류, “다시 시도” 제공. 횟수는 확정 전이면 복구.
  • 미성년: 기본 OFF. 광고 관련 문구 없음.
  • 연령 미확인: switch 비활성, “프로필 확인 후 사용할 수 있어요.”

인터랙션

  • 처음 ON: 소비 규칙을 설명하는 bottom sheet 확인 후 활성화
  • OFF: 즉시 비활성화. 이미 시작한 잡은 취소하지 않고 결과를 검토함에 보관
  • 잔여 행 탭: 한도 상세 bottom sheet
  • “자료를 골라 직접 실행” 탭: B 화면으로 이동

B. 자료선택 · 수동 실행

목적: 이번 1회를 어디에 쓸지 명확하게 선택하게 한다. 그래픽은 선택된 자료를 잇는 작은 별 연결선 하나만 사용한다.

레이아웃 구조

  1. top app bar: “리즈닝 자료 선택”과 유일한 뒤로가기
  2. 잔여 카드: 주간 기본 잔여, 보상 잔여, 월요일 초기화
  3. 소스 그룹 목록: 직접 기록, 카카오톡 내보내기, 캘린더, 건강 데이터, YouTube 시청기록, 가계 CSV
  4. 각 행: 소스 아이콘, 이름, 최근 항목 수, 포함/제외 switch
  5. 선택된 소스 안의 최근 자료 목록. 최대 5개를 체크박스로 선택
  6. 고정 하단 버튼: “선택한 n건 리즈닝”

상태 전수

  • 기본: 최근성이 높은 자료를 제안 선택하되 실행 전 사용자가 확인.
  • 소스 0개: 빈 상태와 “첫 자료 담기”. 실행 버튼 비활성.
  • 선택 0개: “자료를 선택하면 실행할 수 있어요.”
  • 잔여 0: 선택 상태를 보존한 채 F 시트를 표시.
  • 실행 중: 중복 실행 차단 후 D 화면으로 전환.
  • 완료: E 결과 검토로 전환.
  • 오류: 선택을 유지하고 재시도. 예약만 됐으면 예약 복구.
  • 자동 ON: “자동 실행과 별개로 직접 고른 1회를 사용해요.”
  • 자동 OFF: 별도 강조 없음.
  • 미성년: 동일한 선택 UX, 광고 진입만 없음.

인터랙션

  • 소스 switch OFF: 해당 소스 자료 선택 해제
  • 자료 탭: 선택/해제. 5개 초과 시 bottom sheet로 이유 안내
  • 실행 탭: 동일한 수동 command에 idempotency key를 붙여 예약 후 D로 이동
  • 홈 말풍선에서 들어온 경우 추천 자료가 미리 선택되지만 동작은 완전히 동일

C. 홈 · 세컨비 리즈닝 제안

목적: 지금 함께 볼 가치가 있는 새 자료가 있음을 한 문장으로 제안한다. 별자리 골격은 그대로 두고 세컨비 말풍선 하나만 바뀐다.

레이아웃 구조

  1. 기존 홈 별자리와 북극성 위계 유지
  2. 기존 세컨비 head 아래 말풍선: 태그, 한 문장, 최대 2개 action
  3. 노드 위 모달 금지. 상세는 B 화면 또는 F bottom sheet로 이동

상태 전수

  • 기본: 최근 새 자료 수와 주간 잔여를 관찰 톤으로 제안.
  • 자동 ON: 자동 묶음 예정 시간 또는 대기 상태 안내.
  • 자동 OFF: “같이 볼까요?”와 수동 진입.
  • 실행 중: “진행 화면 보기” 단일 action.
  • 완료: “제안 n개가 준비됐어요”와 “결과 검토”.
  • 잔여 0·성인 광고 가능: “광고 보고 2회 받기”, “플랜 보기”.
  • 잔여 0·미성년/연령 미확인: 광고 없음. “플랜 보기”만.
  • 오류: 단정하지 않고 “이번 연결은 마치지 못했어요.”

  • “결과 검토”: E로 이동

목적: 작업이 안전하게 진행 중이며 화면을 떠나도 된다는 사실만 전달한다. 그래픽은 세컨비 head와 일정 속도의 궤도 진행 링 하나다.

  2. 중앙 세컨비 head + 비탄성 궤도 링
  3. 한 줄 상태와 처리 항목 수
  4. “백그라운드에서 계속” primary, “실행 취소” text action
  5. 다른 화면에서는 전역 task strip만 지속 노출

상태 전수

E. 실행 완료 · 결과 검토와 비준

목적: AI의 관찰을 사용자가 선택적으로 비준하게 한다. 그래픽은 북극성이 항상 우세한 작은 별자리 미리보기 하나다.

레이아웃 구조

  1. top app bar “결과 검토”
  2. 요약 1문장과 mini constellation. 북극성 크기·밝기 우위 유지
  3. 제안 목록: 도메인 연결, 밝기 변화, 북극성 문장, 페르소나 서사
  4. 각 제안: 근거 자료 보기, 선택 checkbox, “제안” 라벨
  5. 고정 하단 action: “선택한 n개 반영”

상태 전수

  • 기본: 제안은 기본 선택되지만 사용자가 모두 해제 가능.
  • 선택 0개: CTA “모두 기존대로 두기”.
  • 반영 중: 중복 탭 차단, 각 proposal idempotency 유지.
  • 완료: 선택한 별만 밝기 갱신, 북극성 비준 시에만 북극성 연출.

구현 순서 승인: 말씀하신 대로 화면보다 먼저 리즈닝 실행 잡 인프라를 구현해 주세요. 순서는 예약 → 분석 → proposal 영속화 → 사용량 확정 → 비준이며, 실패·취소·재시작 복구와 idempotency를 먼저 닫은 뒤 A~F를 연결합니다.

필수 구현 계약

  1. 상태 머신: queued → reserved → running → proposed → ratified. 대체 종료는 cancelled / failed / recovered.
  2. Idempotency: 수동 command, 자동 batch, 사용량 확정, proposal 반영에 각각 안정된 key를 둡니다. 재탭·재전송·앱 재시작으로 중복 차감하거나 중복 반영하면 안 됩니다.
  3. 예약과 확정: 실행 가능성을 원자적으로 예약하되, proposal이 영속화되기 전 실패하면 예약을 복구합니다. 횟수 확정 후 결과가 사라지는 상태를 허용하지 않습니다.
  4. 자동 할당: weekly base와 monthly reward ledger를 분리해 조회합니다. 자동 잡은 base에서 수동 1회를 예약하고 reward ledger에는 접근하지 않습니다.
  5. 비준 전 무쓰기: domain tag, brightness, 북극성, persona, source page, Phase 1 frontmatter를 비준 전에 쓰지 않습니다.
  6. 부분 비준: proposal 단위로 exactly-once 반영하고 성공한 항목을 재시도하지 않습니다. 최신 태그를 다시 읽어 동시 편집을 덮어쓰지 않습니다.
  7. 백그라운드 복구: 전역 task 상태와 서버 job 상태를 재연결합니다. 화면이 없어도 완료·오류가 사라지지 않아야 합니다.
  8. 연령 fail-closed: isMinor === false일 때만 광고를 허용합니다. 연령 미확인 상태에서는 광고·LLM 실행 모두 보류합니다.
  9. 동일 수동 command: 설정 자료선택과 홈 말풍선은 같은 command, 같은 한도 시트, 같은 결과 route를 사용합니다.
  10. 디자인: 공통 DeepSpaceScreen, 북극성 우위, 화면당 메시지 1개와 그래픽 1개, bottom sheet 또는 화면 전환, 뒤로가기 1곳을 유지합니다.
  11. 카피·i18n: 위 KO/EN 페어를 locale key로 옮기고 EN↔KO parity 검사를 통과시킵니다.
  12. SAME-QUALITY: 플랜 차이는 횟수·기능·보관 기간뿐입니다. 모델 품질이나 자기 이해의 우열을 암시하지 않습니다.
  13. 잔여 분리 표기: 현재 합산된 “N회 남음 · 월요일 초기화”는 부정확합니다. 주간 기본 잔여와 월간 보상 잔여를 별도 필드와 별도 초기화 시점으로 표시합니다.
  14. 보상 진입점: 현재 홈과 리즈닝의 /records CTA는 실제 RewardedSheet로 이어지지 않습니다. F의 단일 limit sheet가 실제 광고 실행과 원래 선택 상태 복귀를 소유하도록 통합합니다.
  15. 광고 자격: 단순 adsConfigured()만으로 노출하지 않습니다. Free, isMinor === false, 광고 동의, SDK 준비, 허용 surface를 모두 서버와 UI에서 확인합니다.
