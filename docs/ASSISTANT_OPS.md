## 3. 실행(푸시) 로드맵 — 게이트 순서대로

### P1 — JS-only, 게이트 없음 (즉시 가능)
- **ICS(.ics) 생성기**: 추천 일정 → RFC 5545 파일 생성 → 공유시트/다운로드.
  Android·iOS·웹 전부에서 기본 캘린더(삼성/구글/애플)가 .ics를 연다. $0.
- **Google Calendar URL 템플릿**: `calendar.google.com/calendar/render?action=TEMPLATE&...`
  딥링크 — 설치된 구글 캘린더 앱/웹으로 폼이 미리 채워진 채 열림. OAuth 불필요.
- **투두 푸시 v0**: 체크리스트를 공유시트(텍스트)로 — 모든 투두 앱이 수신 가능.

### P2 — 네이티브, EAS 트랙 (코드 준비 → 빌드 게이트)
- `expo-calendar`: **기기 캘린더에 직접 등록** — Samsung Calendar·Google Calendar·
  Apple Calendar 전부 기기 캘린더 프로바이더로 수렴하므로 별도 앱별 연동 불필요.
  이것이 "사용자가 원하는 일정 관리 앱" 커버의 본질 경로.
- `expo-notifications`: 루틴 리마인더(로컬 알림, 서버 푸시 불필요, $0).

### P3 — OAuth/SDK, Simon 콘솔 게이트
- Google Calendar API + **Google Tasks API**(투두) — GCP 콘솔 + OAuth 동의화면.
- **Health Connect**(Android): Samsung Health·InBody·핏빗 등이 쓰기하는 표준 허브.
  개별 앱 SDK 파트너십(삼성헬스 직접 연동 등) 대신 이 허브 하나로 수신 — 현실 경로.
- **HealthKit**(iOS): EAS + 권한 카피. 둘 다 건강 데이터라 **민감정보 — 동의·암호화·
  미성년 잠금 설계 동반(법무 게이트)**.
- Notion·Slack 임포터(O-R2 ⑤-iii 그대로 흡수).

### P4 — 이메일 수신 / 뉴스
- 이메일 전달 주소(in-bound parse) 또는 IMAP 읽기 = 프라이버시 최중량 — 별도 설계서 필수.
- 뉴스 요약: 외부 피드 수급 + 비용 — 후순위.

## 4. 추천 엔진

- 입력: 도메인 선택(온보딩에서 사용자가 고름 — 캡처 UI가 바로 이 선택 화면) +
  해당 도메인 관련 기록/위키/평가 발췌(기존 RAG 스냅샷 계약 재사용, 일기 무유입
  원칙은 export 사이클 21 결정 유지).
- 처리: C9 분류 → C1 gemini.ts(구조화 출력: 제목·이유·제안 일정·반복) → C3 감사.
- 비용: 무료 N회/일 체험 + Soma/Cortex/Brain 단계 캡 — **gemini-proxy 서버 캡(사이클 9)
  재사용**, 신규 purpose 라벨 `ops` 추가.
- 안전: 운동·식단 추천은 의료 조언 아님 고지 1줄(어휘 정책 준수 — 진단·처방 어휘 금지,
  계획·루틴·아이디어 프레이밍). red zone 입력은 기존 위기 플로우 그대로.

## 7. 제약 정합 체크

- C1/C3/C9: 전부 기존 경로 재사용 — 신규 LLM 진입점 없음.
- 무료 티어 $0: P1·P2 전부 $0(로컬 알림·ICS·딥링크). 추천 LLM 비용은 기존 캡 구조.
- 프라이버시: 연동별 opt-in 토글(privacy_prefs 신규 키, 미성년 서버시드 잠금 패턴),
  푸시 건별 승인, 외부 데이터는 사용자가 켠 항목만 수신.
- 어휘 정책: 운동/식단/건강 = 루틴·계획·아이디어 프레이밍. 금지어 회피 카피 가이드
  를 locale 작업 시 포함.
- i18n: 신규 네임스페이스 `ops` — EN 정본, C7 전 로케일 패리티(ES 포함 즉시 적용).
