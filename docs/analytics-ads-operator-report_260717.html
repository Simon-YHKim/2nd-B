개인정보 화면에서 서버 저장이 성공한 뒤에만 사용 통계를 켭니다. 끌 때는 저장 결과를 기다리지 않고 앱 이벤트를 막고 GA consent를 denied로 바꾸며 Clarity consent·쿠키를 철회합니다. 이미 로드된 Google tag의 cookieless ping은 아래 콘솔 전송 차단도 필요합니다.

안전한 GA4 화면 경로

Expo 파일 세그먼트만 사용해 /record/[id]처럼 전송합니다. 모든 GA 이벤트의 page_location, referrer, title을 덮어쓰고 UUID·긴 token·query·hash를 방어적으로 제거합니다.

Clarity·PostHog·Sentry 방어

Clarity ConsentV2에서 광고 storage는 항상 denied입니다. 문서 전체에 Clarity mask 속성을 적용했고 PostHog 자동 URL·referrer는 제거했습니다. Sentry URL·breadcrumb도 익명화하고 tracing을 껐습니다.

계정 전환 비동기 안전성

개인정보 설정 저장 중 로그아웃하거나 다른 계정으로 바뀌어도 이전 사용자의 응답이 새 계정의 화면 상태를 덮지 않도록 대상 user와 mounted 상태를 재검사합니다.

A. GA4·Clarity 운영 검증

[작업] 2nd-Brain 웹 분석 운영 상태를 검증한다. 새 속성이나 프로젝트를 중복 생성하지 않는다.

대상 저장소: Simon-YHKim/2nd-B
운영 URL: https://simon-yhkim.github.io/2nd-B/
원칙: 비밀번호, 2FA, 전체 ID 값을 채팅이나 보고서에 남기지 말고 마스킹한다. 약관 동의나 설정 삭제는 Simon에게 넘긴다.

1. GitHub Actions Variables에서 EXPO_PUBLIC_GA4_MEASUREMENT_ID와 EXPO_PUBLIC_CLARITY_PROJECT_ID의 존재만 확인한다. 값은 출력하지 않는다.
2. GA4 Admin에서 기존 웹 Data Stream URL이 운영 URL과 맞고 Measurement ID가 GitHub 값과 일치하는지 확인한다. Google Signals와 광고 개인화는 켜지 않는다.
3. Enhanced Measurement의 Page views 설정에서 browser history 기반 page changes를 OFF로 둔다. 앱이 수동 page_view를 보내므로 자동 SPA pageview는 중복과 실제 URL 노출 위험이 있다.
4. Google tag > Configure tag settings > Manage data transmission에서 analytics_storage=denied일 때 behavioral analytics data와 consent diagnostics data 전송을 차단한다. 이미 로드된 tag는 consent denied에서도 cookieless ping을 보낼 수 있다.
5. 배포 후 성인 QA 계정으로 사용 통계 OFF 상태를 새로고침한다. googletagmanager.com, google-analytics.com/g/collect, clarity.ms 요청이 없어야 한다.
6. 사용 통계를 ON으로 저장한 뒤 화면을 3곳 이동한다. GA4 Realtime/DebugView에 page_view가 화면 이동당 1건인지 확인한다. 경로는 /record/[id] 같은 파일 패턴이어야 하며 실제 record/user ID, query, 기록 본문은 없어야 한다.
7. Clarity Settings에서 Masking을 Strict로, unmask 예외를 0건으로 둔다. Default cookies는 OFF, Consent Mode는 ON으로 둔다. GA 연동과 광고 공유는 켜지 않는다.
8. 사용 통계를 다시 OFF로 바꾼다. 앱 이벤트가 즉시 멈추고 GA consent가 denied로 바뀌며 Clarity cookies가 지워져야 한다. Network에서 GA automatic/cookieless request가 남으면 전송 차단 설정을 수정하고 PASS 처리하지 않는다.
9. 결과를 PASS/FAIL 표, 시간, 브라우저, 마스킹된 스크린샷으로 보고한다. 새 테스트 계정은 만들지 않는다.

B. Microsoft Clarity 정책 확인 영문 문의

2nd-Brain is a general-audience self-understanding application that supports direct registration for users aged 14-17 as well as adults.

The Microsoft Clarity JavaScript tag is never injected when the authenticated user's server-recorded date of birth indicates an age under 18 or when age is unknown. It is also never injected before explicit optional analytics consent. The tag loads only for authenticated users aged 18 or older who opted in, with Strict masking, a document-wide data-clarity-mask attribute, default cookies disabled, Consent Mode enabled, and advertising storage denied.

Does this implementation comply with Clarity's restriction stating that Clarity should not be used on websites or apps targeting users under 18?

The app also has dynamic paths such as /record/<uuid> and /peer/<token>. Strict text masking does not mask URLs. What supported configuration can prevent Clarity from collecting or retaining those path identifiers without breaking navigation analytics?

Please provide written confirmation and any additional configuration requirements. Until we receive confirmation, we will keep production use disabled.

C. AdSense 사전감사, 활성화 금지

범위가 5파일을 넘으면 native analytics와 AdMob을 각각 별도 PR 계획으로 나누고 첫 슬라이스만 구현해.
