## Cross-cutting hard-safety rules (per `docs/research/CLAUDE.md` §0)

All design-doc decisions inherit these. None of the v0.2 design directly contradicts them. Reminder:

1. Crisis routes to `crisis-detection.md` first — never to analysis system
2. Never claim diagnosis / therapy
3. Never infer protected categories
4. Never use Tier C labels as inference grounds (MI is Tier C → supplementary only per `multiple-intelligences.md`)
5. Never coerce redemptive narratives
6. Rumination interrupt at 3+ revisits in 14 days
7. Always disclose AI mediation

## What's NOT yet covered by this PR

These are noted for future deep-research rounds:

1. **CBT/REBT extension** — `cbt-rebt.md` exists; deeper integration with the lexicon (avoiding clinical claims) is a future cycle
2. **K-MSCEIT** — does not exist in DOI-indexed Korean literature; Korean EI relies on K-WLEIS / K-SEIS (self-report). Documented gap.
3. **K-CRT** — not findable. Substituted with Choi/Koo/Choi 2007 AHS (more culturally calibrated anyway).
4. **PerLLM-Persona-style cross-LLM trait inference benchmark** — no canonical paper exists; the Golden Set Protocol in v0.2 §08 effectively builds one in-house.
5. **Single canonical GPT-4 vs Claude vs Gemini consistency paper** — does not exist; assembled from slice studies (Sclar 2024, Ouyang 2023, Chen-Zaharia-Zou 2023).
6. **Indigenous psychology beyond East Asian** — Latin American, African, South Asian frameworks. Future batches.

## How to extend this map

When v0.3 of the design doc adds new claims:

1. Find or curate the backing research (new batch via `psychology-handoff.md`)
2. Add a row to the relevant § section above
3. Update `batches/INDEX.md` totals
4. If a claim **lacks** verifiable backing, document it here as a gap rather than fabricating
5. If a new finding **contradicts** v0.2, flag it explicitly so the design doc can be revised
