1 · Brain Trinity — 출처 방법론 SOURCE

docs/research/batches/methodology-birkman-brain-trinity.md — "Self-Awareness Methodology Manual v1.0 (2026.05.25)" 비판적 감사. 저자 "브레인 트리니티의 브라이언"(Brian Choi, 한국 유튜브).

측정 → 설계 → 운영 파이프라인

측정
Birkman Method (1:1 컨설턴트 필수). 9개 구성요소 중 유일하게 동료심사 검증(Maxwell 2016, 중간 수준)
설계
Brain Trinity 4계층: 0th Brain(Polaris 철학문서/5W1H) · 1st Brain(Tiago Forte PARA in Obsidian) · 2nd Brain(workspace 툴링) · 3rd Brain(Karpathy LLM Wiki 패턴)
운영
일/주/월/년 사이클로 네 계층을 묶음

감사 결론: 9개 중 Birkman만 peer-review(중간), 나머지(PARA·Karpathy Wiki·Polaris·Zettelkasten·DIKW·GTD·Bush 1945)는 실무/역사 출처. 매뉴얼이 전부 동등 검증된 듯 제시 → 오해소지. 유지할 것: 측정-먼저-설계 순서 · 살아있는 문서(v0.1→v1.0) · Karpathy Wiki(LLM 통합). 과장: Birkman 편향 Big Five 비교표(실제 Big Five는 직무·수명 예측 강함).

앱의 현행 /trinity = 이 개념의 후손 — Simon 본인 운영 4영역(건강/앱/뇌/재정) 인생관리 대시보드(태그 집계). BUILT 단, 위 4계층(0th~3rd)과는 다른 단순화 버전.

§00 매니페스토

§02 LLM-Agnostic 4 Layer

Layer내용상태
L1 사실 카드JSON 구조화 입력(constrained decoding) + RAG 그라운딩부분 (phase1/wiki)
L2 이론 매트릭스인지(CHC) · 성격(Big Five+landscape) · 동기(SDT) · 정서(EI/MSCEIT) · 인지양식(dual-process) · 보조(Gardner MI Tier C)대부분 설계만
L3 행동 분류Behavior Taxonomy(행동 코딩, κ 신뢰도)설계만
L4 Cross-LLM 검증비결정성 대비 교차검증설계만

핵심 프로토콜

  • Ledger vs Voice 분리 — cold 사실(Ledger) / warm 재서술(Voice, 사실 추가 불가). 부분 BUILT (persona/center.ts · self-portrait.ts · chat/personas.ts)
  • 3-Tier 증거 저장 — Tier A 원문 영구 / B 맥락 30일 / C 사용자선택+purge (GDPR 17). 설계
  • Ambiguity Resolution Queue — 저신뢰 항목을 앱 진입 시 사용자 보정(active learning). 설계
  • Golden Set Protocol — 분기별 100 페르소나 κ 캘리브레이션, dimension std-dev → taxonomy 개정. 설계
  • Develop Queue Q1~Q5 — Output Schema · Behavior Taxonomy 마스터 · 이론 매트릭스 · Golden Set · lexicon+법무. 설계
