- **agy headless 실기 QA 신뢰성 미검증**: 이 머신 헤드리스에서 에뮬 설치+캡처 풀사이클 실증된 바 없음(agy --print은 파일읽기 텍스트만 안정, 디렉터리스캔/write 풀사이클 무출력 종료 이력). **07-04 AG "에뮬 통과" 3건이 픽셀 없이 날조→무효**된 전례.
- **픽셀 판정은 어차피 hub 몫**(ROUTING 정본: Claude가 스크린샷 Read로 픽셀 직독 재판정). agy가 캡처해도 hub가 판정. 중간 단계(agy)가 날조 위험만 추가.
- **에뮬 가용 타이밍**: 지금 에뮬 살아있음(자주 죽음). 즉시 hub 수행이 최선.
→ hub가 adb로 직접 픽셀 QA 수행. 증거 = 실제 스크린샷 파일 경로 + Read 픽셀 직독.

## 항목 1 — 서명 불일치 (uninstall-first) → **PASS** (정적 실증, 에뮬 불필요)
- 방법: `apksigner verify --print-certs` 로 rc2 vs v0.0.9 릴리즈 APK 서명자 대조.
- 결과: **동일 서명자** — 둘 다 `V2 Signer SHA-256 = 0fb37bc076c46bc45a637c04bacfce7613bb0db91eda21b74bce595ad00ce570`.
- 판정: rc2와 기존 릴리즈가 **같은 EAS 관리 서명 키** → 기존 v0.0.9 설치본 위에 rc2 덮어쓰기 가능. **uninstall-first 불필요.**
- 주의: gradle 진단 APK(throwaway keystore)와는 서명이 다르나, 그건 릴리즈 경로 아님(diagnostic only). 사용자/QA가 설치할 EAS APK끼리는 동일 서명.
- → UNRESOLVED "서명 불일치" **해소**.

## 항목 2·3 — 광고 게이트 실기 + 픽셀 (진행 중)

### 항목 1 — 서명 불일치 → **PASS** (정적 실증)
- rc2 vs v0.0.9 서명자 동일(`0fb37bc0...`). uninstall-first **불필요**. (에뮬 불필요, 위 §항목1 참조.)

### 항목 2 — 광고 게이트 fail-closed → **설계 PASS / 실기 UNVERIFIABLE**

### 항목 3 — 픽셀 직독 → **UNVERIFIABLE** (앱 크래시, 단 원인 특정됨)
- 앱 실행 → **"2nd-Brain keeps stopping"** 크래시 다이얼로그 픽셀 확인(`docs/qa/rc2-260721/02-app.png`).
- logcat: `com.facebook.soloader couldn't find DSO to load: libreactnative.so`.
- **원인 = ABI 미스매치(에뮬 환경 아티팩트), APK 결함 아님**: rc2 APK 네이티브 lib 29개 **전부 arm64-v8a**, x86_64 0개(eas.json `ANDROID_ABI_FILTER: arm64-v8a`, 148MB 축소 의도). 에뮬은 x86_64 → libreactnative.so 로드 실패. **실기(모든 현대 폰=arm64)에선 정상.**
- 이 머신에 **arm64 시스템 이미지 없음** → x86_64 에뮬만 가용 → 실기 확증 불가. 확증엔 **실기 arm64 폰** 필요.

## QA 프로세스 인계 (실측)
- **arm64-only preview APK는 x86_64 에뮬에서 설치되나 실행 크래시**(soloader/libreactnative.so). 향후 에뮬 QA는 (a) arm64 시스템 이미지 AVD 또는 (b) x86_64 포함 빌드 프로파일 필요. 실기 QA는 arm64라 무관. (메모리 native-delivery-gap 재확인.)
- 에뮬 안정성 낮음: 최초 `emulator-5554` shell hang, `2ndB_QA_009` 콜드부트 offline, Pixel만 부팅 성공. QA 착수 전 에뮬 실응답(shell PONG) 확인 필수.
