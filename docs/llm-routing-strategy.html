2nd-Brain · Engineering Note

상황·시스템별 적정 LLM 라우팅 전략

"리즈닝이 필요한 곳엔 강한 모델, 속도가 필요한 곳엔 빠른 모델 + 하네스로 성능을 메운다" — 이 가설의 검증과, 실제 우리 코드(14개 PromptPurpose) 기준 매핑.

결론: 가설이 맞습니다. 그리고 우리는 이미 그 방향으로 절반 가 있습니다 — 코드의 MODELS는 flash(gemini-2.5-flash) / pro(gemini-2.5-pro) 2-tier고, Advisor(저널 성찰)만 Pro, 나머지는 전부 Flash입니다. 남은 건 이걸 purpose별 3-tier 라우터로 공식화하고 모델 세대를 2026 라인업으로 올리는 것뿐입니다.

1. 핵심 원리 — 하네스가 메울 수 있는 것 vs 없는 것

"빠른 모델 + 하네스"가 통하는지는 출력의 성격이 가릅니다. 하네스(검색 그라운딩·스키마 강제·분류기 선게이트·검증/재시도·human propose→ratify)는 회상·형식·안전을 대신해 줄 수 있지만, 판단·종합(synthesis)은 대신 못 합니다.

하네스로 커버됨 → 빠른 모델

모델은 "유능하고 빠르면" 충분. 정답이 구조화·검증 가능.

  • 분류 / 태깅 / 라우팅 (정답 집합이 닫혀 있음)
  • 추출 / OCR / 전사 (입력에 답이 이미 있음)
  • 검색 기반 응답 (RAG가 근거를 떠먹여 줌)
  • 스키마 강제 출력 (JSON 강제 디코딩)

하네스로 못 메움 → 리즈닝 모델

출력이 모호한 개인 데이터에 대한 판단. 검색·검증으로 환원 불가.

  • 북극성(Soul Core) 정체성 문장 종합
  • 반복측정 인터뷰의 다음 질문 설계(미묘한 정합성↑)
  • 저널 성찰 / propose→ratify 변경 제안
  • 기록 간 숨은 연결·패턴 발견

한 줄 규칙 — 출력이 "라벨/추출/경로"면 빠른 모델, "사용자가 누구인가에 대한 통찰"이면 리즈닝 모델.

3. 제안 — purpose별 3-tier 라우터

현 2-tier(flash/pro)를 3-tier로 확장하고 callGemini({ purpose })가 purpose→tier를 자동 선택하게 합니다. (지금은 호출부가 직접 model을 넘김.)

Tier S · 초고속/최저가

대상: 안전분류(C9)·capture/clipper 분류·임베딩 — 모든 호출에 붙는 최고 볼륨·최고 지연민감 경로.

모델: gemini-2.5-flash-lite 급 (출시 라인업 중 최저가 Flash 계열) · 임베딩은 text-embedding-004 유지.
하네스: lexicon 선필터가 대부분을 LLM 전에 거름 → 모델은 회색지대만. batch 50% + 캐시 90%로 단가 바닥.

Tier F · 균형 (기본값)

대상: 세컨비 챗·OCR·음성 전사·ops 추천·지식조회·임포트 — 사용자가 기다리는 인터랙티브 경로.

모델: gemini-3.5-flash (2.5-flash에서 업그레이드 — 3.1 Pro 대비 ~4× 속도에 코딩/에이전트 벤치는 상회, $1.50/$9.00, 캐시 $0.15).
하네스: RAG 검색 그라운딩 + 근거칩 인용 + 스키마 강제 출력이 품질을 끌어올림.

Tier R · 리즈닝

대상: Advisor 저널 성찰·인터뷰 질문 설계·북극성 종합·연결 발견·공상 구체화 — "사용자가 누구인가"를 판단.

모델: gemini-3.x-pro (현 2.5-pro 업그레이드). 볼륨이 낮아 단가 부담 작음 — 여기서 비용 아끼면 제품의 핵심 가치(자기이해 깊이)가 깎임.
하네스로 대체 불가. 단, propose→ratify의 human 게이트가 "틀린 제안"의 비용을 흡수하므로 risk는 관리됨.

4. 왜 GPT/Claude가 아니라 Gemini 유지인가

티어(우리가 쓸 급)입력/출력 1M무료티어오디오 입력비고
Gemini 3.5 Flash$1.50 / $9.00있음네이티브이미 통합(C1/C2/C3), 1M 컨텍스트
GPT-5.4 Mini / Nano$0.75/$4.50 · $0.20/$1.25없음별도 경로단가는 최저, 무료티어·통합 부재
Claude Haiku 4.5$1.00 / $5.00없음불가KO 우수하나 오디오 전사 불가

GPT Nano가 토큰 단가만 보면 가장 싸지만, 우리 맥락에선 (1) 무료티어로 $0/mo를 지키고 (2) 방금 붙인 오디오 전사가 단일 API로 끝나며 (3) C1 경계가 이미 Gemini 기준이라 — Gemini가 종합 우위. 비용 폴백이 정말 필요해지면 Tier S(대량 분류)만 GPT-5.4 Nano로 빼는 하이브리드가 가능하지만, 복잡도 대비 실익은 쿼터 초과 전엔 작습니다.
