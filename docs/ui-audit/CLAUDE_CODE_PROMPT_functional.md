# 2nd-Brain · 기능 동작 점검 (Claude Cowork `/goal` 자율 모드)

> 목표: UI가 "보이는가"가 아니라 **실제로 동작하는가**를 점검한다. 버튼·입력·저장·
> 네비게이션·AI 응답·결제·동의 게이트·propose→ratify·동기화가 의도대로 작동하는지
> 검증하고, 깨진 기능을 심각도순으로 정리한 뒤 가능한 것을 고친다. 완료 정의가 전부
> 참이 될 때까지 멈추지 마라. 새 기능 발명 금지 — 기존 동작의 정상성만 검증/수리.

## 0. 점검 방법 (가능한 것 택1, 병행 권장)
### A. 자동 E2E (가장 강력)
- 헤드리스 브라우저로 실제 클릭/입력/이동을 스크립트한다(라이브 또는 로컬 expo web).
- 라이브 base https://simon-yhkim.github.io/2nd-B 는 TLS 프록시 때문에 헤드리스가
  ERR_CERT_AUTHORITY_INVALID로 거부한다 → 컨텍스트에 `ignoreHTTPSErrors:true`,
  launch args `--no-sandbox`, Chromium '/opt/pw-browsers/chromium-1194/chrome-linux/chrome',
  Playwright(CommonJS) `import pkg from '/opt/node22/lib/node_modules/playwright/index.js'`.
  goto waitUntil:'networkidle' + 4초 대기, 뷰포트 390×844 isMobile. (로컬이면 인증서 무관)
- 테스트 계정으로 로그인해 auth-gated 플로우도 통과시켜라(없으면 신규 가입 플로우로 생성).
### B. 유닛/통합 테스트
- npm test (또는 jest). 기존 테스트 통과 여부 + 핵심 로직(엔진/스토어/리듀서) 커버리지 확인.
- 비어 있는 핵심 경로엔 회귀 방지용 테스트를 추가해도 좋다(기존 컨벤션 따라).
### C. 정적 추적 (캡처/실행 불가 시 최소선)
- 라우트 등록·핸들러 연결·스토어 액션·async 호출부를 코드로 따라가 "죽은 핸들러/
  미연결 네비게이션/never-resolved Promise"를 찾는다.

## 1. 점검할 기능 흐름 (FUNCTION_CHECKLIST.md 전부)
각 항목: 입력 → 기대 결과 → 실제 결과(OK/실패/부분) → 콘솔/네트워크 에러 → 재현 경로.

### 인증·세션
- 로그인(정상/오자격) · 회원가입(미성년 DOB → 통신·위치 임포트 잠금 실제 적용) ·
  비번 재설정 메일 · OAuth 콜백 → /complete-profile 강제 게이트 · **로그아웃 → /sign-in** ·
  미인증으로 보호 라우트 접근 시 /sign-in 리다이렉트 · 프로필 없으면 /complete-profile 강제.

### 네비게이션 무결성
- 모든 진입점(셸 4-primary·하단탭·별·설정 행)이 의도한 라우트로 이동(죽은 버튼 0).
- Back은 한 곳에서 직전 화면으로. 중복 진입점 없음. 노드/카드 위 모달 없음(시트/화면전환).

### 입력·저장·동기화
- /capture 5모드(글·링크·사진·음성·할 일) 저장 → /records·/record/[id]에 반영.
- 오프라인에서 담기 → 큐 적재 → 온라인 복귀 시 자동 동기화(거짓 성공 없음).
- /record/[id] 편집·이동·삭제가 실제 데이터에 반영(삭제 후 리스트에서 사라짐).
- /inbox triage(태그확정/보관/삭제)로 "남은 N개" 감소.

### 자기이해 엔진
- /interview 반복 질문 → 응답 제출 → 다음 질문 진행 → 종료 시 렌즈(Big Five 등) 갱신이
  propose→ratify로만 반영(자동 확정 없음). AI 실패 시 재시도 경로.
- 7별 밝기(L1~L5)·북극성 종합이 입력에 따라 갱신. /ttfv 비준 시 별 L1→L2 실제 상승.

### 비서 Ops
- /ops 도메인 선택 → 추천 카드 생성(행동이력 반영). 완료 체크 → 연속일수 반영.
- 추천 "캘린더에 추가/공유" → 푸시 시트(①기기캘린더 ②Google ③.ics ④체크리스트) →
  최초 1회 동의(ops_push) 없으면 동의 먼저. 웹에서 기기캘린더 숨김. **자동 실행 없음.**
- 예약 리마인더 등록·끄기·상태칩(활성/권한필요/이 기기 불가)이 실제 권한과 일치(거짓 성공 0).
- /focus 타이머 시작·일시정지·리셋·완료가 집중 루틴 기록에 반영.

### 데이터 임포트·내보내기·연동
- /import 소스별 동의 시트(무엇을/어디에/이 기기에서만) → 파일 선택/권한 → 파싱
  ProgressBar → 결과 요약 → 검토 후 반영(propose→ratify). 미성년 통신·위치 서버 잠금.
- /formats 형식 선택+범위 토글 → 내보내기 산출물 생성. /iden .iden 생성·열람.
- /integrations 커넥터 연결/해제 OAuth 왕복.

### 성장·공유·결제
- /growth 주간 집계가 실제 데이터(star_tier_history·ops 로그)에서 산출. 다음 한 걸음 →
  /ops·/milestones로 ratify 전달. /imagine 공상 → 첫 걸음 후보 떨어뜨리기.
- 공유 카드 동적 생성(핸들·문장 주입) → 이미지 캡처·공유 시트 호출.
- 페이월: 티어(별바라기/항해자/북극성) 구매 플로우(IAP) · "구매 복원" · 엔티틀먼트에 따라
  횟수/렌즈/보관 한도 실제 적용. 리워드 광고 시청 → +N 크레딧 실제 증가(품질 불변).

### 로딩·전역 오버레이
- 긴 작업 startTask(background) → BackgroundTaskDock 표시 → 완료 시 CompletionToast,
  "결과 보기"로만 이동(자동 이동 없음). 앱은 분석 중에도 계속 조작 가능.

### i18n·접근성
- KO/EN 전환 시 키 누락·깨짐 없음(check:i18n). 터치타깃 ≥44px. OS 폰트 스케일 반영.

## 2. 산출물
- docs/func-audit/REPORT.md: 흐름별 (입력·기대·실제·에러·재현경로).
- **심각도 우선순위표**: blocker(핵심 플로우 불능·데이터 유실·거짓 성공·결제 실패·동의
  게이트 우회) / major(부분 실패·죽은 버튼·동기화 누락·propose→ratify 우회) / minor(엣지).
  각 항목에 "어느 파일/함수, 어떻게 고칠지".
- 재현 스크립트/테스트는 docs/func-audit/ 에 저장.

## 3. 고치기 (가능한 범위)
- blocker·major 중 명확한 것은 고친다(미연결 네비게이션·죽은 핸들러·자동 실행 누락
  게이트·거짓 성공 토스트·never-resolved Promise). 고친 것마다 npm run type-check +
  관련 테스트 통과. 이미 잘 도는 로직·비주얼 티어 시스템은 임의 리팩터 금지(회귀 0).
- 안전: 결제·삭제·임포트 같은 위험 동작은 실제 과금/영구삭제가 일어나지 않도록 테스트
  계정·샌드박스·드라이런으로. 사용자 데이터 파괴 금지.
