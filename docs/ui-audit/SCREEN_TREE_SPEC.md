# 2nd-Brain · 전체 화면 트리 / 인터랙션 / 네비게이션 정본

> Claude Code 구현용 단일 명세. **시각**의 출처는 design/*.dc.html, **동작·네비게이션**의
> 출처는 이 문서다. 이미 레포에 구현된 화면이 대부분이며, 이 문서는 "각 화면이 무엇을
> 하고 / 어떤 버튼이 어디로 가고 / 어떤 상태를 갖는가"의 기준이다. 충돌 시 이 문서 +
> design 정본을 따른다. 새 화면·새 토큰·off-palette 발명 금지.

## 표기
- `route` = expo-router 경로. `→` = 이동(navigation). `⤺` = 뒤로/닫기.
- 각 화면: **역할** / **진입(어디서)** / **버튼·액션→목적지** / **인터랙션** / **4상태**.
- 공통 규율: deepSpace.* 토큰만, ≥44px 터치, 한 화면=한 메시지+한 그래픽,
  propose→ratify(자동 반영 없음), i18n EN/KO. 큰 세컨비 머리는 터치 추적(자동, size≥80).

# 0. 진입 셸 & 전역 네비게이션

## 0.1 셸 구조 (모든 1차 화면 공통)
- **상단 상태헤더**(SecondbStatusHeader): 좌측 작은 세컨비(추적 안 함) + 우측 말풍선
  (현황 1문장 + TIP). 화면마다 문구 다름.
- **하단 탭바**(PremiumTabBar): 1차 라우트에서만 노출. 4 탭 = **담기 / 알아가기 / 비서 /
  나**(+ 중앙 세컨비). 탭 = 단일 진입(중복 금지, F5 dedup).
- **BackArrow**: 2차 화면 좌상단 한 곳에만. ⤺ 직전 화면.
- **전역 오버레이**: BackgroundTaskDock(하단, 백그라운드 작업 중) · CompletionToast(상단, 완료).
- **큰 세컨비 머리**: SecondbHeadTrackProvider 하위에서 터치를 추적(둥둥+표정+감정 오브).

## 0.2 진입점 맵 (어디서 무엇으로)
- 셸 4-primary: **담기**→/capture · **알아가기**→/index(별자리 홈) · **비서**→/ops ·
  **나**→/account.
- 별자리 홈(/index): 북극성→/core-brain, 북두칠성 7별→각 렌즈(아래 2장).
- 더보기/설정 진입: /settings → manual·theme·plans·permissions·privacy·support·**museum**·integrations.
- 콜드스타트: 폰트 로드(InlineLoader) → IntroGate(인증·인트로) → 미인증=/sign-in,
  인증·프로필無=/complete-profile, 정상=/index.

# 1. 인증 · 온보딩

## /sign-in — 로그인
역할: 이메일/소셜 로그인. 진입: 콜드스타트(미인증)·로그아웃 후.
버튼: [로그인]→/index · [회원가입]→/sign-up · [비밀번호 재설정]→/reset-password ·
[소셜(Google/Apple)]→/oauth-callback→/index. 인터랙션: 이메일·비번 검증, 에러 인라인,
비번 표시 토글(EyeIcon). 4상태: 빈(폼)·로딩(제출 중 버튼 디밍)·에러(자격 실패 토스트)·채움.

## /sign-up — 회원가입
역할: 계정 생성 + 생년월일(미성년 C10 게이트) + 동의. 진입: /sign-in.
버튼: [가입]→(성공)→/complete-profile · [이미 계정 있음]→/sign-in. 인터랙션: DOB 입력으로
미성년 판정→통신·위치 임포트 서버잠금 고지, 약관 동의 토글. 4상태: 빈·로딩·에러(중복 이메일)·채움.

## /reset-password — 비밀번호 재설정
역할: 재설정 메일 발송. 진입: /sign-in. 버튼: [메일 보내기]→확인 안내 · ⤺/sign-in.
4상태: 빈·로딩·에러(미가입 이메일)·발송완료.

## /complete-profile — 프로필 완성
역할: OAuth 후 공개 프로필·동의 확정(이게 없으면 어떤 기능도 Gemini 호출 불가→강제 게이트).
진입: 인증+프로필無 자동 리다이렉트. 버튼: [시작하기]→/onboarding(또는 /index).
인터랙션: 닉네임·언어·핵심 동의. 4상태: 빈·로딩·에러·채움.

## /onboarding (+ /jot) — 첫 입력
역할: 최소 입력 1개로 자기이해 씨앗 만들기. 진입: complete-profile 직후.
버튼: [첫 기록 담기](/jot 인라인)→[다음]→/ttfv. 인터랙션: 한 질문 한 화면, 건너뛰기 허용.
4상태: 빈(프롬프트)·작성중·에러·완료.

## /ttfv — 첫날 자기이해 한 컷  (design/ttfv-firstday.dc.html)
역할: 30초 아하 모먼트. 북극성 Tier1 + 한 별 Tier2 점등 + 통찰 1줄.
진입: 온보딩 직후 1회. 버튼: [맞아요](ratify→그 별 L1→L2)·[조금 달라요](보정) · (탭 후)
[이 별 더 알아가기]→해당 렌즈. 인터랙션: 별 fade+bloom 점등, 비준 시 밝아짐.
4상태: 제시·비준후(근거 노출)·첫주(데이터 적음 안내)·에러.

# 2. 알아가기 — 북극성 + 북두칠성 7별  (design/lens.dc.html, core-brain.tsx)

## /index — 별자리 홈 (딥스페이스)
역할: 앱의 얼굴. 상단 별자리(북극성 12시 + 북두칠성), 하단 큰 세컨비 머리(터치 추적)+말풍선.
진입: 셸 "알아가기" / 콜드스타트. 버튼: 북극성→/core-brain · 7별 각각→해당 렌즈 ·
하단 말풍선(소개/팁/요약/할일/미션 회전). 인터랙션: 머리가 터치 추적, 별 탭 시 머리가
그 별을 보고 말풍선 전환, 감정 오브 디밍(긍/중/부). 4상태: 빈(첫 진입)·로딩·에러·채움.

## /core-brain — 북극성 · 소울 코어
역할: 7별의 종합 = 나의 북극성 문장(철학/정체성). 입력 버튼 없음(종합이라). 진입: 홈 북극성.
버튼: [근거 보기](7별 기여 펼침) · ⤺. 인터랙션: 보라 오브 + 7별 종합 밝기. 4상태:
빈(별 부족 시 "더 알아가야 또렷")·로딩·에러·채움.

## 7별 렌즈 (공통 패턴)
각 별 = 측정된 상태 화면 + 데이터 입력 버튼([다시 입력]/[데이터 추가]). 진입: 홈 별 탭/ttfv.
공통 4상태: 빈(미측정 "첫 측정 시작")·로딩·에러·채움(그래픽+신뢰도 L1~L5).

- **/persona (지금의 나 · Big Five)** → Brain-Trinity식 AI 반복 질문 도구.
  버튼: [다시 검사]·[측정 이어가기]→/interview. 그래픽: Big Five 5축 바 + L레벨.
- **/big-five** — Big Five 상세/결과. 버튼: [재검사]→/interview.
- **/mbti** — 유형 보조 렌즈. 버튼: [검사]→/interview.
- **회상(recall)** — 시기별 바둑판 그리드(유아기~지금). 버튼: 시기 칸→drill-down 질문
  (/interview 컨텍스트). 그래픽: 타임라인. (라우트: persona/회상 또는 audit 내 섹션)
- **/attachment (관계 · 애착)** — 애착 유형 렌즈. 버튼: [다시 입력]→/interview.
- **/esm (리듬)** — 경험표집(하루 신호). 버튼: [지금 기록]→ESM 프롬프트. 그래픽: 리듬 차트.
- **보여지는 나(peer)** — 남이 본 나. 버튼: [설문지 보기]·[공유로 peer review 요청]→공유시트.

## /interview — AI 반복 측정 인터뷰
역할: 한 번에 확정 않고 교묘히 반복되는 질문으로 정합성↑(성격검사 방식). 진입: 각 렌즈 입력 버튼.
버튼: [답변 제출]→다음 질문 · [그만하기]→직전 렌즈(부분 저장). 인터랙션: AI가 핵심을 되물음,
진행 바. 4상태: 빈(시작)·진행·에러(AI 실패 재시도)·완료(렌즈 갱신 propose→ratify).

## /audit — 라이프 오딧
역할: 삶의 영역 점검·평가 종합. 진입: 알아가기/나. 버튼: 영역→drill-down. 4상태 동일.

# 3. 매일 허브  (design/hub.dc.html, capture.tsx)

## /capture — 담기
역할: 매일 입력의 핵심. 5모드(글·링크·사진·음성·할 일) 입력 + 최근 조각. 진입: 셸 "담기".
버튼: 모드 칩→입력 전환 · [담기]→저장(최근에 추가) · 최근 항목→/record/[id]. 인터랙션:
오프라인에서도 담기(동기화 대기 표시). 4상태: 빈("첫 기록 담기")·로딩·에러(저장 실패 큐)·채움.

## /secondb — 세컨비 (대화)
역할: 내 데이터 위에서 답하는 개인 AI. 진입: 셸 중앙/허브. 버튼: [전송]→답변(근거 칩) ·
근거 칩→출처 /record/[id] · [후속 제안]→해당 화면 · [공상 모드]→/imagine. 인터랙션: 머리
아바타 챗, 인용 표시. 4상태: 빈(웰컴)·생성중·에러·대화.

## /jarvis — 세컨비 풀 어시스턴트
역할: secondb의 확장(작업 지향). 진입: 허브/secondb. 버튼: 제안 실행→대상 화면.

## /trends (트렌드) — 관심 상승
역할: 최근 관심사 상승 감지·제안. 진입: 허브. 버튼: 제안 카드 [담기]→/capture 컨텍스트.
4상태: 빈·로딩·에러·채움.

## /review (점검) — propose→ratify 승인
역할: 세컨비가 제안한 변경(태그·연결·해석)을 사용자가 승인. 진입: 허브/알림.
버튼: [승인](diff 반영)·[수정]·[거절]. 인터랙션: before/after diff. 4상태: 빈(대기 없음)·로딩·에러·대기N.

## /insights — 인사이트 (주간)
역할: 주간 WoW 변화 요약. 진입: 허브. 버튼: 항목→근거/렌즈. 4상태: 빈(첫주)·로딩·에러·채움.

## /digest — 다이제스트
역할: 기간 요약 묶음. 진입: 허브/인사이트. 버튼: 항목→상세.

/record/[id] · 세컨비 한 줄의 📎근거→렌즈 · [편집]·[이동(정리)]·[삭제(위험 톤)]. 인터랙션:

[태그확정]·[보관]→/records·[삭제]. 인터랙션: "남은 N개" 진행감. 4상태: 빈("정리 끝!")·로딩·에러·채움.

## /research — 연결 찾기

## /wiki — 지식 (RAG 읽기면)

## /journal — 저널

# 5. 개인 비서 Ops  (design/ops-ia.dc.html, ops/screens.tsx)

## /ops — 비서 추천 홈
역할: 도메인 피커(5그룹, Hick≤5) → 추천 카드 리스트 + "오늘의 루틴". 진입: 셸 "비서".
버튼: 도메인 칩→전용 화면 또는 공통 추천 · 추천 카드 [저장/보내기](푸시 시트)·[공유] ·
[예약 리마인더]→리마인더. 인터랙션: 완료 탭(햅틱+스케일+연속일수). 4상태: 빈·로딩·미연결·채움.

## 14영역 → 6화면 라우팅 (전용 5 + 공통)

- **/milestones (커리어·학습목표)** — 마일스톤 리스트(상태칩 전환, 마감경고)+진행률, 주기 회고.

# 10. 네비게이션 계약(요약 규칙)
- 한 기능 = 한 진입점(중복 금지). Back은 한 곳(BackArrow). 노드/카드 위 모달 금지(시트/화면전환).
- 한 터치는 화면을 단순화(드릴다운/바텀시트). 자동 실행·자동 반영 없음(propose→ratify).
- 미인증→/sign-in, 인증+프로필無→/complete-profile 전역 게이트. 미성년 통신·위치 임포트 잠금.
- 큰 세컨비 머리는 모든 화면에서 터치 추적(자동). 작은 헤더 머리는 추적 안 함.
