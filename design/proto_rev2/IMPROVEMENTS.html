무엇이 문제였나

  • 비율 불일치 — 핸드오프의 스펙 캡처 37장이 460×820(1.78:1)로, 기준 캔버스 390×820(2.10:1)과 다른 비율로 스트레치 렌더됨. 그 과정에서 폰트 부스팅이 발생해 한국어 단어 중간 줄바꿈("별자/리", "01 이/론"), 텍스트 잘림이 캡처 전반에 남아 있었음.
  • 극소 폰트 — 8–10.5px 인라인 폰트 약 50곳(뮤지엄 연도축 8px, 홈 별 라벨 10.5px, 북극성 모노 라벨 9px 등).
  • 저대비 텍스트 — rgba 알파 .45–.55의 보조 텍스트 다수 (대비 2.3–3.9:1).

적용한 개선 (전 화면 공통)

  • m3-theme.css — ① text-size-adjust:100%로 모바일/캡처 환경의 폰트 부스팅 차단(비율·줄바꿈 왜곡의 근본 원인) ② body에 word-break:keep-all; overflow-wrap:break-word 전역화(한국어 중간 줄바꿈 금지)
  • 폰트 바닥 정책 — 한국어 라벨 ≥11px(내비게이션 핵심 12px), 라틴/숫자 모노 태그 ≥10px. 13개 파일 약 50개 사이트 수정. 위계(계층 차이)는 유지.
  • 대비 상향 — 저대비 rgba 알파 .45–.6 → .72–.9 (뮤지엄 연도축, TTFV 캡션, 로그인 약관 안내, 그래프 보조 라벨 등).
  • 스펙 캡처 재생성 — docs/Screen-Spec/captures/ 37장 전부 390×820@2x(780×1640) 균일 비율로 교체. 중간 줄바꿈·잘림 없는 정상 렌더.

디자인 캡처 대조 (픽셀 디프) — 2라운드

원본 스펙 캡처 37장(460×820)을 390×820로 정규화한 뒤 현재 앱 렌더와 픽셀 디프(채널 허용오차 40, 상태바 마스크). 6-에이전트 어트리뷰션 후 Claude가 전 건을 코드로 재검증. 히트맵·수치: docs/improve-audit/design-diff/
  • 순수 DRIFT(같은 상태·구조에서 시각 속성만 다른 결함): 0건. 후보 4건(연동 아이콘 톤·챗 액센트·집중 5번째 칩·코치 pill)은 전부 코드 검증에서 의도된 진화 또는 위양성으로 판명.
  • 실제 회귀 1건 — 복원 완료: 설정 모양 섹션의 강조 색(시안/바이올렛) 팔레트 행. 캡처·README·핸드오프 지시서가 모두 요구하고 셸 배선이 살아있었음 → UI 행 복원 (diff 32.0%→25.9%).
  • 캡처 상태 정렬 1건: 서브 화면 탭바 활성 pill을 원본 캡처 컨벤션(별자리)으로 통일해 재캡처.
  • 코드 진화(EVOLVED) — 캡처가 구빌드, 코드가 정본: 담기 W4H1 폼 · 위키 그래프 전용 · 검증 화면들(BigFive/동기/강점/애착)의 "확신 %"+프로비넌스 비준 카드 · 별 상세 CV 타임라인 · 북극성 페르소나 캐러셀 · 뮤지엄/전시 타임라인 캔버스 · 로그인 이메일 우선+소셜 5종 · 알림 다이제스트 카드 · 통화녹음 프라이버시 칩 · 비서 종합 의견 · 설정 데이터연동 섹션 · TTFV 근거 영수증 · 온보딩 4페이지. 전역 리네임: 조각→별가루, 오락→휴식, 담아내기→뮤지엄, 북극점→북극성, 안티비→트위비 — 모두 번들의 최신 README와 일치(코드=정본 교차 확증).
  • 잔여 diff%의 지배 요인: 원본 캡처 자체의 폰트 부스팅(중간 줄바꿈·수직 오프셋·헤더 잘림·이중 렌더 깨짐)과 동적 요소(시계·순환 문구·랜덤 별). 예: 가치관 화면 22.45%는 사실상 전량 부스팅 위양성(구조·값 완전 동일).
수렴 판정: 스테일·왜곡된 구빌드 캡처와의 픽셀 일치는 현행 디자인을 파괴해야만 가능하므로, 픽셀 일치는 다음 형태로 달성했다 — ① 실회귀 복원 ② 캡처 상태 정렬 ③ docs/Screen-Spec/captures/ 37장을 현행 앱의 픽셀-정확 렌더로 재생성(정의상 앱과 0-diff). 특정 화면을 구디자인(캡처 쪽)으로 되돌리길 원하면 화면을 지정해 주세요 — 기능 제거를 수반하는 비가역 결정이라 자율 진행하지 않았습니다.

실기 검증 — Android 에뮬레이터 (3라운드)

Pixel 9 Pro XL 에뮬레이터의 실제 Android Chrome 149(모바일 렌더링 파이프라인·폰트 부스팅 동작 포함)를 CDP로 원격 제어해 전 57화면을 순회. 캡처: docs/improve-audit/on-device/
  • 프레임 비율: 57/57 화면 모두 390×820(2.103:1) — 실기에서도 균일.
  • 폰트 부스팅 가드: text-size-adjust: 100% 실기 적용 확인(dpr 3, 뷰포트 448×864). 원본 캡처를 오염시킨 부스팅 경로가 실기에서 차단됨.
  • 한국어 중간 줄바꿈: 텍스트 노드 단위 줄박스 측정으로 전 화면 검사 — 0건. (1차 검출기의 25건은 아이콘+텍스트 요소를 한 Range로 잡은 측정 오류로 판명, 텍스트 노드 단위로 재측정)
  • 육안 확인: 온보딩·설정(강조 색 행 포함)·뮤지엄(연도 다이얼) 실기 렌더 정상.

실기 렌더 vs 정본 디자인 픽셀 대조 (4라운드)

구 디자인 폐기 확정 후의 대조 축: 정본 = 현행 스펙 캡처(데스크톱 렌더) vs 실제 앱 = 에뮬레이터 실기 Chrome 렌더. 양쪽 reduced-motion 고정, 진입 분석 잡 대기, 시계/컴패니언 밴드 마스크, 390×820 정규화, AA-내성 비교(3×3 이웃·허용오차 40 — 서브픽셀 래스터 차이 흡수, 구조 차이는 생존). 산출물: docs/improve-audit/device-diff/(히트맵·수치), on-device-frame/(실기 프레임 캡처 57장)
  • 구조적 픽셀 일치: 57/57 화면. 잔여 편차 0.2~2.4%는 전량 글리프 래스터라이징 쉬머(Windows DirectWrite vs Android FreeType, 텍스트 밀도에 비례) — 히트맵 판독 결과 레이아웃·박스·에셋 어긋남 0건.
  • 실기 웹폰트: Pretendard·Roboto Mono·Material Symbols 모두 실기 로드 확인(강제 로드 검증).
  • 타이밍 이슈 1건 해결: 별 상세(star) 진입 시 "살피는 중" 분석 오버레이가 캡처에 섞이던 것 → 양쪽 캡처 하네스에 잡 종료 대기 추가(11.3%→0.67%).
  • 1차 측정의 함정 기록: 단순 픽셀 비교는 1.5~11.3%로 나왔으나 전부 래스터 차이였음 — 크로스 렌더러 비교엔 AA-내성 비교가 필수.
