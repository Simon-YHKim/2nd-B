명명·통합 단계에 도달해 있다.

핵심 엔진은 이미 배포되어 있다. BFI-44, ECR-S, 5x5 인터뷰, 페르소나 합성, Gemini 하네스, 안전 분류기, 클리퍼와 위키 RAG, SecondB 챗 모두 살아있다. 남은 일은 새 시스템을 짓는 것이 아니라, 흩어진 척도를 하나로 묶고, 별 7개를 같은 종류로 정의하고, 측정의 정지규칙을 부여하고, 용어를 정본화하는 것이다.

이 문서가 답하는 것

부족한 것

측정의 정지규칙(언제 그만 묻나), 별 밝기 = 데이터 품질의 정의, 별6·별7의 정체, 보여지는 나(peer), 트렌드 제안, propose-ratify, ESM 연결. 4단계 두뇌 모델과 용어가 문서에만 있고 코드 표면엔 미반영.

보완된 모델

북극성 1개 + 자기이해 렌즈 7별(역할/액션/지식은 별이 아닌 목표트리로 강등), 두뇌 0/1/2/3, 단일 L1~L5 가치 사다리(DIKW·Bloom·밝기·신뢰도 통합), 측정 하네스(경량 신뢰도 + 삼각측정 + self-other), 용어 정본 매핑, propose→ratify 신뢰 루프.

화면별 구조

42개 라우트를 북극성 별자리 홈 1개 + 7별 + 5허브로 정렬. 한 화면 = 한 메시지 + 한 그래픽. 새 화면은 보여지는 나·리듬·미래·트렌드제안·점검리뷰 5개뿐, 나머지는 유지/수정.

한 문단 요지 당신이 가진 가장 강한 카드는 새 기능이 아니라 이미 배포된 검증 엔진 하나를 끝까지 잇는 것이다. 지금의 나(BFI-44) → 데이터 품질 점수 → 별 밝기 → 그래프 → 비서가 그 위에서 답함 → 비서가 다음 학습 한 걸음을 제안(propose-ratify). 이 단일 루프가 11항목 전체의 논지("AI가 나를 데이터로 축적해 전용 비서가 되어 북극성을 향하게 한다")를 증명한다. 나머지는 "설계됨, 일부만 시연"으로 정직하게 표시한다.

2 부족한 것 (Gaps) - 심각도별

네 개 렌즈와 비평이 합의한 누락 지점이다. BLOCKER 는 먼저 결정하지 않으면 이후 작업이 막히는 것, MAJOR 는 시연 품질을 좌우하는 것, MINOR 는 구현 시 챙길 것이다.

2.1 차단급 (먼저 결정해야 코드가 안 엉킨다)

심각도갭왜 차단인가적용한 해결(이 문서가 채택)
BLOCKER 7별의 정체가 렌즈마다 다르다 + "지금의 나"가 별인지 대시보드인지 모호 어떤 별은 '내가 맡은 역할', 어떤 별은 '나를 아는 방법' - 종류가 섞이면 밝기 척도와 점수법을 공유할 수 없다. 이것이 당신이 느낀 혼란의 뿌리. 역할/액션/지식을 별에서 강등(북극성 목표트리로). 7별 = 모두 자기이해 렌즈 한 종류. 지금의 나 = 별1(도구), 소울 코어 = 7별을 종합한 판독값(별 아님). (3a 참조)
BLOCKER 풀 IRT/CAT는 솔로 2개월에 과설계 bfi.ts는 단순 평균 채점, item 변별도(a)·난이도(b)·θ 추정기 전무. BFI-44는 IRT 뱅크로 설계되지 않아 공개 파라미터도 표준화돼 있지 않음. 여기에 매달리면 런웨이 소진. CAT/IRT는 post-XPRIZE로 강등. v1은 결정론적 경량 신뢰도 프록시(완성도+관찰수+최신성+교차합의)로 대체. θ→밝기는 서사적 목표로만 유지. (3d 참조)
BLOCKER 보여지는 나(peer)가 C10/PIPA와 충돌 peer는 제2의 행위자(타인) 데이터를 저장하고, peer가 미성년일 수 있고, 사용자가 14-17 미성년일 수 있다. DPIA-minors 초안과 정면 충돌. 동의 설계 없이 출시 = 법적 차단. XPRIZE 윈도에서는 성인 한정·동의 게이트·미성년 실데이터 미수집 mock 또는 완전 후순위. SOKA 방법론은 타당하나 동의 게이트를 넘지 못함. (5번·6번 참조)

2.2 주요급 (시연 품질을 좌우)

  • MAJOR 별 밝기 = 데이터 품질의 구체 공식이 없다. 네 렌즈 모두 "신뢰도/SE/수렴에서 도출하라"고만 하고 계산 가능한 v1 공식을 안 줬다. 이 문서가 3c·3d에서 오늘 계산 가능한 한 공식을 못박는다.
  • MAJOR 인터뷰 정지규칙 부재. probe.ts는 커버리지(5층 1회씩 터치) 또는 50턴 캡으로 종료 - 측정된 신뢰도로 멈추지 않는다. 비전 1·2의 "신뢰도 높은 점수 도달까지 deep-dive"가 미증명. 경량 신뢰도 임계 + 수렴(삼각측정)으로 교체.
  • MAJOR C1 하네스의 "도구 vs LLM 분리"가 미명문화. 현재 휴리스틱 점수는 이미 결정론적 TS다(LLM은 서사 요약만 작성). 위험은 "현재 채점"이 아니라 미래의 자유텍스트 추출 경로에 있다. 그 경로를 짓기 전에 스키마+검증기+계약테스트를 둔다.
  • MAJOR /persona ↔ /core-brain 병합은 라벨 변경이 아니라 리팩토링. buildCenterCards와 buildPersona는 별개 로직 - 합칠 때 회귀 위험을 '리팩토링(회귀위험)'으로 표시.

2.3 경미급 (구현 시 챙길 것)

  • MINOR 임상어휘 가드. 'validated' 단독은 안전(lexicon에 없음). 그러나 clinically validated / 임상적으로 검증된 / diagnosis / treatment / cure / healing 은 하드밴드. L4 라벨은 "교차검증(cross-source agreement)"으로. 새 문서·UI엔 check-forbidden-lexicon.ts 실행.
  • MINOR em dash 금지. 제안된 모든 one_message·인사이트 문자열은 구현 시 하이픈+공백으로. check-emdash.ts 실행.
  • MINOR ESM 연결은 "작지만 신규". build.ts에 ESM은 주석으로만 존재 - '2줄 연결'이 아니라 상태(state) vs 특성(trait) 매핑 결정이 필요. 시간이 없으면 XPRIZE 표면에서 깔끔히 제거가 더 안전.

3 보완된 통합 모델

여기가 메모의 심장이다. 흩어진 것을 하나의 정합한 구조로 묶는다. 핵심 원칙: 한 종류의 별, 한 종류의 밝기, 한 번도 LLM이 점수를 발명하지 않는 하네스.

3a. 북극성 + 북두칠성 7별 (최종안)

근본 수정은 레벨 혼합 제거다. 역할(Roles)·액션(Action)·지식(Knowledge)은 철학 분해 트리의 가지(북극성 아래 Who/Why/What → Roles → Projects → Tasks)이지 측정축이 아니다. 이들은 trait 추정치도, 신뢰도도, 밝기도 만들지 않는다. 따라서 별에서 빼서 북극성 목표트리로 렌더한다. 7별은 오직 자기이해 렌즈 - 각각 (a)추정하는 잠재구성, (b)elicitation 경로, (c)스코어러, (d)밝기 판독값을 갖는다.

"지금의 나"는 별이지 대시보드가 아니다 지금의 나 = 별1(검증 도구로 측정하는 trait state). 소울 코어/페르소나 카드 = 7별 전부를 받아 종합하는 판독값(aggregate) = 북극성 밝기의 표시기. 별이 아니다. 이 구분을 캡션이 아니라 정본 문장으로 못박는다.

3b. 두뇌 0/1/2/3 모델 + 앱의 역할

정본 한 문장으로 못박는다: "2nd-Brain은 당신의 2nd brain(데이터의 유기적 망)을 짓고 3rd brain(AI 비서)을 배선해 1st brain(물리 뇌)을 0th brain의 북극성(철학)을 향해 정진시키는 다리(bridge)다." 앱은 결코 '두뇌'가 아니라 다리/비계다. 각 층을 이미 정본인 3축에 바인딩해 떠다니지 않게 한다.

3c. 단일 가치 사다리 L1~L5 (DIKW · Bloom · 밝기 · 신뢰도 통합)

떠다니던 다섯 척도(DIKW / Bloom / 밝기5단계 / 데이터품질5단계 / 신뢰도)를 하나의 순서척도로 붕괴시킨다. 이 하나의 정수가 동시에 노드·별 밝기 = 데이터 품질 = 출처 신뢰도 = 드릴 정지레벨이다. 노드는 다음 증거 조건이 충족될 때만 올라간다(태깅→연결→교차검증→실행). AI는 상승을 제안하고 사용자가 승인한다.

오늘 계산 가능한 v1 밝기 공식 (IRT 불필요) 축별 품질 티어 = 증거 경로들 중 최대값으로: 검증도구 완료 → 티어 4~5 · 저널/감사 관찰 ≥15 → 티어 4 · 5~14 → 티어 3 · 1~4 → 티어 2 · 기본 → 티어 1, 그리고 ≥2 독립 경로가 일치하면(교차수렴) +1 티어, 최대 5에서 상한. 이는 traitConfidenceFor()를 그대로 재사용하며 오늘 계산된다. 별 밝기와 다음행동 넛지가 같은 정의를 공유하게 된다.

3d. 측정 / 하네스 (lean v1)

핵심 down-scope: 풀 IRT/CAT(Fisher information, θ 추정기, 정적 a/b 파라미터)는 솔로 2개월에 과설계이며 코드베이스에 없는 item 수준 데이터를 요구한다. 따라서 CAT/IRT는 post-XPRIZE로 미루고, 정지규칙과 점수를 lean하게 푼다.

고정 도구 (BFI/ECR) - 경량 신뢰도

θ·SE·정지를 결정론적 프록시로: 신뢰도 = f(도구 완성도 + 관찰수 + 최신성 + 교차경로 합의). 적응형 item 순서는 단순 변동/커버리지 휴리스틱으로 충분. SE→밝기는 서사적 목표로만 유지(v1 산출물 아님). 풀 Fisher CAT은 아님.

자유텍스트 AI 질문 - 삼각측정/수렴

보정된 item 뱅크가 없으므로 SE 대신 수렴 기준: 한 셀(시기×층)을 그 신호가 (a)고정도구 추정치, (b)다른 elicitation 경로(ESM·peer·다른 시기)와 일치할 때까지 프로브. ≥2 독립 경로 일치(삼각측정) OR 추정치가 더 안 움직임(포화) = 충분. 질적 연구의 이론적 포화의 정량 아날로그.

self-other 검증 (Vazire SOKA) - 보여지는 나는 기능이 아니라 타당도 계층 자기(self)는 내적·저관찰성·저평가성 특성(신경증, 애착불안, 사적 감정)의 더 나은 판단자, 타인(other)은 외적·고관찰성·평가성 특성(외향성, 특히 지성/개방성)의 더 나은 판단자. 따라서 특성별 가중: 신경증/애착불안 → self(별1,4,5) 우세, 외향성/지성·개방성 → peer(별3) 동등 이상. 특성별 self-other 합의 점수를 계산 → 높은 합의 = 높은 타당도 = 더 밝은 별. 큰 불일치 = 성장 인사이트로 표면화("남들은 너의 X를 너보다 높게 본다", 비임상 프레이밍).
LLM vs 도구 분리 - "어떤 AI든 고품질" 의 구체·검증 가능한 형태 INSTRUMENT 층(결정론·코드·LLM 비의존): 검증 item 뱅크, 구성 정의, 역코딩, 스코어러 - 이것만이 trait 점수를 방출한다(이미 bfi/ecr에 존재). LLM 층(gemini.ts 경유, C1): 두 가지 일만 - (1)다음 질문을 자연스럽게 표현, (2)자유텍스트를 {construct_id, direction, strength, quote} 타입 증거로 추출. responseSchema + 검증기가 LLM이 내놓은 숫자 점수를 거부. 계약 테스트: 모의 LLM이 쓰레기 점수를 반환하는 고정 트랜스크립트를 넣어 페르소나 점수가 불변임을 단언. 비고: 현재 휴리스틱 채점은 이미 결정론적 TS이므로 "현재 코드를 뜯어내라"가 아니라 "이 속성을 보존하고 자유텍스트 추출 경로를 짓기 전에 가드를 둬라". 새 LLM 경로엔 C3 감사 + C9 선분류 필수.
